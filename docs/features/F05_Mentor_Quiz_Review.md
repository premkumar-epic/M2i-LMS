# 1. Feature Overview

## 1.1 What Is This Feature?

The Mentor Quiz Review Interface is the quality control gate between
AI-generated quiz questions and student-facing assessments. It is a
dedicated workflow and interface that allows mentors to efficiently
review, approve, edit, or reject AI-generated quiz questions before
they are made available to students.

While Feature 04 handles the generation of questions, Feature 05 is
entirely about the mentor's experience of reviewing those questions.
The two features are closely related but serve completely different
purposes. Feature 04 is a background AI pipeline. Feature 05 is a
human-facing workflow tool.

This feature exists for one critical reason: AI-generated content is
not perfect. Language models occasionally produce questions with subtle
factual errors, ambiguous wording, incorrect distractors, or questions
that do not align with the actual learning objectives of the content.
The mentor review step catches these issues before they reach students
and corrupt the learning experience or the metrics data.

The design philosophy of this interface is speed without sacrificing
quality. A mentor reviewing 20 generated questions should be able to
complete the review in under 10 minutes. Every design decision — the
layout of the question card, the one-click approve button, the inline
edit form, the batch approve option — is made with this time efficiency
goal in mind.

## 1.2 What This Feature Does

**Review Queue Management:** Provides a consolidated view of all
pending quiz questions across all content items in the mentor's batch.
The mentor sees everything that needs their attention in one place,
not scattered across individual content pages.

**Question-Level Actions:** For each generated question, the mentor
can: approve it as-is with one click, edit the question or options
and then approve, or reject it with a categorized reason.

**Batch Operations:** Mentors can approve all currently visible
questions at once when a content item has produced consistently high
quality questions.

**Progress Tracking:** The interface shows a running count of pending,
approved, and rejected questions so mentors always know how much work
remains.

**Quality Analytics:** Provides visibility into approval rates per
content item, helping identify patterns in AI generation quality and
indicating when prompt improvements are needed.

**Minimum Threshold Alerts:** Notifies mentors when a content item
has fewer than the minimum required approved questions to release the
quiz to students, prompting manual question creation if needed.

## 1.3 Relationship to Other Features

This feature is directly downstream of Feature 04 (AI Quiz Generation)
and directly upstream of Feature 07 (Quiz Taking System). No question
generated by Feature 04 reaches students in Feature 07 without passing
through this review interface.

The rejection data this feature collects feeds back into Feature 04's
prompt improvement process — high rejection rates for specific reason
categories indicate specific prompt weaknesses that need addressing.

# 3. Data Model

## 3.1 No New Tables

Feature 05 does not introduce new database tables. It operates
entirely on the quizzes table introduced in Feature 04. The key
fields from that table that this feature uses are:

```
generation_status    : PENDING_REVIEW → APPROVED or REJECTED
approved_by          : mentor's user ID, set on approval
approved_at          : timestamp of approval
rejection_reason     : controlled vocabulary string, set on rejection
original_question_text: stores pre-edit version on first edit
original_options     : stores pre-edit options on first edit
was_edited           : flag set to TRUE on first edit
```

## 3.2 New Index for Review Queue Performance

The review queue query fetches all PENDING_REVIEW questions for a
batch. Add a specific index for this query pattern:

```sql
-- Already defined in Feature 04 schema but important to note here
CREATE INDEX idx_quizzes_batch_pending
  ON quizzes(batch_id, generation_status, created_at)
  WHERE generation_status = 'PENDING_REVIEW';
```

This partial index only covers PENDING_REVIEW rows — as questions are
approved or rejected, they leave this index, keeping it small and fast.

## 3.3 QuizReviewSessions Table (Optional Analytics)

This optional table tracks review sessions for analytics — how long
mentors spend reviewing, which questions require the most time, and
so on. Implement if analytics on review behavior are needed.

```sql
CREATE TABLE quiz_review_sessions (
  id                    UUID        PRIMARY KEY
                                    DEFAULT gen_random_uuid(),
  mentor_id             UUID        NOT NULL
                                    REFERENCES users(id),
  batch_id              UUID        NOT NULL
                                    REFERENCES batches(id),
  content_id            UUID        REFERENCES content(id),
  session_started_at    TIMESTAMP   NOT NULL DEFAULT NOW(),
  session_ended_at      TIMESTAMP   DEFAULT NULL,
  questions_reviewed    INTEGER     DEFAULT 0,
  questions_approved    INTEGER     DEFAULT 0,
  questions_rejected    INTEGER     DEFAULT 0,
  questions_edited      INTEGER     DEFAULT 0
);

CREATE INDEX idx_review_sessions_mentor
  ON quiz_review_sessions(mentor_id, session_started_at DESC);

CREATE INDEX idx_review_sessions_batch
  ON quiz_review_sessions(batch_id);
```

## 4.2 Individual Question Action Endpoints

### POST /api/quizzes/:quizId/approve

**Access:** MENTOR (assigned to batch), ADMIN, SUPER_ADMIN

**Purpose:** Approve a single quiz question

**Request:** No body required.

**Business Rules:**

- Question must be in PENDING_REVIEW status
- Mentor must be assigned to the batch that owns this question
- Sets generation_status = APPROVED, approved_by, approved_at

**Success Response (200 OK):**

```json
{
  "success": true,
  "data": {
    "quiz_id": "quiz-uuid-1",
    "generation_status": "APPROVED",
    "approved_by": "mentor-uuid-1",
    "approved_at": "2026-03-21T14:30:00Z",
    "content_threshold_status": {
      "quiz_type": "QUICK_ASSESSMENT",
      "approved_count": 6,
      "minimum_required": 5,
      "threshold_met": true
    }
  },
  "message": "Question approved"
}
```

**Note on content_threshold_status:** After every approval, the
response includes the updated threshold status for the quiz type this
question belongs to. The frontend uses this to show or hide the
threshold warning banner in real time.

**Validation Rules:**

```
question_text       : optional, string, min 10 chars, max 500 chars
options             : optional, array of exactly 4 strings,

correct_option_index: optional, integer, 0-3
cognitive_level     : optional, one of RECALL, COMPREHENSION,
```

**Behavior:**

- Preserves original AI text in original_question_text

### POST /api/quizzes/:quizId/revoke-approval

**Access:** MENTOR (assigned to batch), ADMIN, SUPER_ADMIN

**Purpose:** Revoke approval of a question — move it back to
PENDING_REVIEW if a mentor realizes a previously approved question
has an error. This is a safety mechanism.

**Request:** No body required.

**Business Rules:**

- Cannot revoke if students have already answered this question
- If students have answered, the mentor must use the edit flow
  instead and create a corrected version

**Success Response (200 OK):**

```json
{
  "success": true,
  "data": {
    "quiz_id": "quiz-uuid-1",
    "generation_status": "PENDING_REVIEW",
    "approved_by": null,
    "approved_at": null
  },
  "message": "Approval revoked. Question returned to review queue."
}
```

**Error Response:**

```json
// 400 - Students have already answered this question
{
  "success": false,
  "error": {
    "code": "CANNOT_REVOKE_ANSWERED_QUESTION",
    "message": "This question has been answered by 12 students 
                and cannot be revoked. Use the edit function to 
                create a corrected version.",
    "details": {
      "response_count": 12
    }
  }
}
```

## 5.3 useReviewQueue Hook

This hook manages the complete review queue state — fetching questions,
handling actions, and keeping the UI in sync with real-time changes
as questions are approved, edited, or rejected.

```typescript
// hooks/useReviewQueue.ts
"use client";

import { useState, useEffect, useCallback } from "react";
import api from "@/lib/api";
import {
  ReviewQuizQuestion,
  ContentGroupSummary,
  BatchSummary,
} from "@/types/quiz.types";

type ReviewQueueState = {
  questions: ReviewQuizQuestion[];
  contentGroups: ContentGroupSummary[];
  batchSummary: BatchSummary | null;
  isLoading: boolean;
  error: string | null;
  currentPage: number;
  totalPages: number;
};

type FilterState = {
  content_id?: string;
  quiz_type?: string;
  cognitive_level?: string;
};

export function useReviewQueue(batchId: string) {
  const [state, setState] = useState<ReviewQueueState>({
    questions: [],
    contentGroups: [],
    batchSummary: null,
    isLoading: true,
    error: null,
    currentPage: 1,
    totalPages: 1,
  });

  const [filters, setFilters] = useState<FilterState>({});

  const fetchQueue = useCallback(
    async (page = 1) => {
      setState((prev) => ({ ...prev, isLoading: true, error: null }));

      try {
        const params = new URLSearchParams({
          page: String(page),
          limit: "30",
          ...filters,
        });

        const response = await api.get(
          `/api/batches/${batchId}/quizzes/review-queue?${params}`
        );

        const { data } = response.data;

        setState({
          questions: data.questions,
          contentGroups: data.content_groups,
          batchSummary: data.batch_summary,
          isLoading: false,
          error: null,
          currentPage: page,
          totalPages: response.data.pagination.total_pages,
        });
      } catch (err: any) {
        setState((prev) => ({
          ...prev,
          isLoading: false,
          error:
            err.response?.data?.error?.message ??
            "Failed to load review queue",
        }));
      }
    },
    [batchId, filters]
  );

  useEffect(() => {
    fetchQueue(1);
  }, [fetchQueue]);

  // Remove a question from the local state after action
  const removeQuestion = useCallback((quizId: string) => {
    setState((prev) => ({
      ...prev,
      questions: prev.questions.filter((q) => q.quiz_id !== quizId),
      batchSummary: prev.batchSummary
        ? {
            ...prev.batchSummary,
            total_pending: prev.batchSummary.total_pending - 1,
          }
        : null,
    }));
  }, []);

  // Approve a question
  const approveQuestion = useCallback(
    async (quizId: string) => {
      const response = await api.post(
        `/api/quizzes/${quizId}/approve`
      );
      removeQuestion(quizId);
      return response.data.data.content_threshold_status;
    },
    [removeQuestion]
  );

  // Reject a question
  const rejectQuestion = useCallback(
    async (quizId: string, reason: string) => {
      const response = await api.post(
        `/api/quizzes/${quizId}/reject`,
        { reason }
      );
      removeQuestion(quizId);
      return response.data.data.content_threshold_status;
    },
    [removeQuestion]
  );

  // Edit a question
  const editQuestion = useCallback(
    async (
      quizId: string,
      edits: {
        question_text?: string;
        options?: string[];
        correct_option_index?: number;
        cognitive_level?: string;
        difficulty?: string;
        explanation?: string;
      }
    ) => {
      const response = await api.put(
        `/api/quizzes/${quizId}`,
        edits
      );
      // Update question in local state with edited version
      setState((prev) => ({
        ...prev,
        questions: prev.questions.map((q) =>
          q.quiz_id === quizId
            ? {
                ...q,
                ...response.data.data,
                was_edited: true,
              }
            : q
        ),
      }));
      return response.data.data;
    },
    []
  );

  // Batch approve
  const batchApprove = useCallback(
    async (quizIds: string[]) => {
      const response = await api.post("/api/quizzes/batch-approve", {
        quiz_ids: quizIds,
      });

      // Remove all approved questions from local state
      setState((prev) => ({
        ...prev,
        questions: prev.questions.filter(
          (q) => !quizIds.includes(q.quiz_id)
        ),
        batchSummary: prev.batchSummary
          ? {
              ...prev.batchSummary,
              total_pending:
                prev.batchSummary.total_pending - quizIds.length,
            }
          : null,
      }));

      return response.data.data;
    },
    []
  );

  return {
    ...state,
    filters,
    setFilters,
    fetchQueue,
    approveQuestion,
    rejectQuestion,
    editQuestion,
    batchApprove,
  };
}
```

## 5.6 ContentGroupSection Component

Groups related questions under their content item heading with
batch approve and threshold alert controls.

```tsx
// components/review/ContentGroupSection.tsx
"use client";

import { useState } from "react";
import QuizQuestionCard from "./QuizQuestionCard";
import ThresholdAlert from "./ThresholdAlert";
import { ContentGroupSummary, ReviewQuizQuestion } from
  "@/types/quiz.types";

type Props = {
  group: ContentGroupSummary;
  questions: ReviewQuizQuestion[];
  onApprove: (quizId: string) => Promise<any>;
  onReject: (quizId: string, reason: string) => Promise<any>;
  onEdit: (quizId: string, edits: any) => Promise<any>;
  onBatchApprove: (quizIds: string[]) => Promise<any>;
  onCreateManual: (contentId: string, quizType: string) => void;
};

export default function ContentGroupSection({
  group,
  questions,
  onApprove,
  onReject,
  onEdit,
  onBatchApprove,
  onCreateManual,
}: Props) {
  const [isBatchApproving, setIsBatchApproving] = useState(false);
  const [batchApproveConfirm, setBatchApproveConfirm] = useState(false);

  const groupQuestions = questions.filter(
    (q) => q.content_id === group.content_id
  );

  const handleBatchApprove = async () => {
    if (!batchApproveConfirm) {
      setBatchApproveConfirm(true);
      return;
    }

    setIsBatchApproving(true);
    setBatchApproveConfirm(false);

    try {
      await onBatchApprove(groupQuestions.map((q) => q.quiz_id));
    } finally {
      setIsBatchApproving(false);
    }
  };

  if (groupQuestions.length === 0) return null;

  return (
    <div style={{ marginBottom: "3rem" }}>
      {/* Content Group Header */}
      <div
        style={{
          display: "flex",
          justifyContent: "space-between",
          alignItems: "center",
          padding: "12px 16px",
          background: "#F9FAFB",
          borderRadius: "8px",
          border: "1px solid #E5E7EB",
          marginBottom: "1rem",
        }}
      >
        <div>
          <h3 style={{ fontSize: "16px", fontWeight: 500, margin: 0 }}>
            {group.content_title}
          </h3>
          <p style={{ fontSize: "12px", color: "#6B7280", marginTop: "2px" }}>
            {groupQuestions.length} question
            {groupQuestions.length !== 1 ? "s" : ""} pending review
            {" · "}
            {group.approved_count} approved
            {" · "}
            {group.rejected_count} rejected
          </p>
        </div>

        {/* Batch Approve Button */}
        {groupQuestions.length > 0 && (
          <button
            onClick={handleBatchApprove}
            disabled={isBatchApproving}
            style={{
              padding: "8px 16px",
              borderRadius: "6px",
              border: "1px solid #D1D5DB",
              background: batchApproveConfirm ? "#059669" : "#F9FAFB",
              color: batchApproveConfirm ? "white" : "#374151",
              fontSize: "13px",
              fontWeight: 500,
              cursor: "pointer",
              transition: "all 0.15s",
            }}
          >
            {isBatchApproving
              ? "Approving..."
              : batchApproveConfirm
              ? `Confirm Approve All ${groupQuestions.length}`
              : `Approve All ${groupQuestions.length}`}
          </button>
        )}
      </div>

      {/* Threshold Alerts */}
      {!group.quick_assessment.threshold_met && (
        <ThresholdAlert
          quizType="QUICK_ASSESSMENT"
          approvedCount={group.quick_assessment.approved}
          minimumRequired={group.quick_assessment.minimum_required}
          contentId={group.content_id}
          onCreateManual={onCreateManual}
        />
      )}

      {!group.retention.threshold_met && (
        <ThresholdAlert
          quizType="RETENTION"
          approvedCount={group.retention.approved}
          minimumRequired={group.retention.minimum_required}
          contentId={group.content_id}
          onCreateManual={onCreateManual}
        />
      )}

      {/* Questions */}
      <div style={{ marginTop: "1rem" }}>
        {groupQuestions.map((question) => (
          <QuizQuestionCard
            key={question.quiz_id}
            quiz={question}
            onApproved={onApprove}
            onRejected={onReject}
            onEdited={onEdit}
          />
        ))}
      </div>
    </div>
  );
}
```

```typescript
// services/quizReview.service.ts
import { PrismaClient } from "@prisma/client";

const prisma = new PrismaClient();

const QUICK_ASSESSMENT_MINIMUM = 5;
const RETENTION_MINIMUM = 8;

export class QuizReviewService {

  // -------------------------------------------------------
  // GET REVIEW QUEUE
  // -------------------------------------------------------
  async getReviewQueue(
    batchId: string,
    mentorId: string,
    filters: {
      content_id?: string;
      quiz_type?: string;
      cognitive_level?: string;
    },
    pagination: { page: number; limit: number }
  ) {
    // Verify mentor is assigned to this batch
    await this.verifyMentorBatchAccess(batchId, mentorId);

    const whereClause: any = {
      batchId,
      generationStatus: "PENDING_REVIEW",
    };

    if (filters.content_id) {
      whereClause.contentId = filters.content_id;
    }
    if (filters.quiz_type) {
      whereClause.quizType = filters.quiz_type;
    }
    if (filters.cognitive_level) {
      whereClause.cognitiveLevel = filters.cognitive_level;
    }

    const [questions, total] = await Promise.all([
      prisma.quiz.findMany({
        where: whereClause,
        orderBy: [
          { contentId: "asc" },
          { quizType: "asc" },
          { createdAt: "asc" },
        ],
        skip: (pagination.page - 1) * pagination.limit,
        take: pagination.limit,
        include: {
          content: {
            select: { title: true, isPublished: true },
          },
        },
      }),
      prisma.quiz.count({ where: whereClause }),
    ]);

    // Get batch summary counts
    const [pendingCount, approvedCount, rejectedCount] =
      await Promise.all([
        prisma.quiz.count({
          where: { batchId, generationStatus: "PENDING_REVIEW" },
        }),
        prisma.quiz.count({
          where: { batchId, generationStatus: "APPROVED" },
        }),
        prisma.quiz.count({
          where: { batchId, generationStatus: "REJECTED" },
        }),
      ]);

    // Get content group summaries
    const contentGroups = await this.getContentGroupSummaries(batchId);

    const formattedQuestions = questions.map((q) => ({
      quiz_id: q.id,
      content_id: q.contentId,
      content_title: q.content.title,
      content_published: q.content.isPublished,
      quiz_type: q.quizType,
      question_text: q.questionText,
      options: q.options as string[],
      correct_option_index: q.correctOptionIndex,
      explanation: q.explanation,
      cognitive_level: q.cognitiveLevel,
      difficulty: q.difficulty,
      source_concept: q.sourceConcept,
      transcript_reference: q.transcriptReference,
      is_ai_generated: q.isAiGenerated,
      was_edited: q.wasEdited,
      original_question_text: q.originalQuestionText,
      created_at: q.createdAt,
    }));

    return {
      batch_summary: {
        total_pending: pendingCount,
        total_approved: approvedCount,
        total_rejected: rejectedCount,
        content_items_with_pending: contentGroups.filter(
          (g) => g.pending_count > 0
        ).length,
      },
      content_groups: contentGroups,
      questions: formattedQuestions,
      pagination: {
        page: pagination.page,
        limit: pagination.limit,
        total,
        total_pages: Math.ceil(total / pagination.limit),
      },
    };
  }

  // -------------------------------------------------------
  // APPROVE QUIZ
  // -------------------------------------------------------
  async approveQuiz(quizId: string, approvedBy: string) {
    const quiz = await this.findAndVerifyQuiz(quizId, approvedBy);

    if (quiz.generationStatus === "APPROVED") {
      throw {
        code: "ALREADY_APPROVED",
        message: "This question is already approved",
        statusCode: 400,
      };
    }

    await prisma.quiz.update({
      where: { id: quizId },
      data: {
        generationStatus: "APPROVED",
        approvedBy,
        approvedAt: new Date(),
      },
    });

    // Return threshold status
    const thresholdStatus = await this.getThresholdStatus(
      quiz.contentId,
      quiz.quizType
    );

    return {
      quiz_id: quizId,
      generation_status: "APPROVED",
      approved_by: approvedBy,
      approved_at: new Date(),
      content_threshold_status: thresholdStatus,
    };
  }

  // -------------------------------------------------------
  // BATCH APPROVE
  // -------------------------------------------------------
  async batchApprove(quizIds: string[], approvedBy: string) {
    const now = new Date();
    const approved = [];
    const failed = [];

    // Process in chunks of 10 to avoid overwhelming the database
    const chunks = this.chunkArray(quizIds, 10);

    for (const chunk of chunks) {
      const results = await Promise.allSettled(
        chunk.map(async (quizId) => {
          await prisma.quiz.update({
            where: {
              id: quizId,
              generationStatus: "PENDING_REVIEW",
            },
            data: {
              generationStatus: "APPROVED",
              approvedBy,
              approvedAt: now,
            },
          });
          return quizId;
        })
      );

      for (const result of results) {
        if (result.status === "fulfilled") {
          approved.push(result.value);
        } else {
          failed.push(result.reason);
        }
      }
    }

    // Get threshold updates for all affected content items
    const affectedQuizzes = await prisma.quiz.findMany({
      where: { id: { in: approved } },
      select: { contentId: true, quizType: true },
      distinct: ["contentId", "quizType"],
    });

    const thresholdUpdates = await Promise.all(
      affectedQuizzes.map(async (q) =>
        this.getThresholdStatus(q.contentId, q.quizType)
      )
    );

    return {
      approved_count: approved.length,
      skipped_count: 0,
      failed_count: failed.length,
      threshold_updates: thresholdUpdates,
    };
  }

  // -------------------------------------------------------
  // REJECT QUIZ
  // -------------------------------------------------------
  async rejectQuiz(
    quizId: string,
    reason: string,
    rejectedBy: string
  ) {
    const VALID_REASONS = [
      "FACTUALLY_INCORRECT",
      "POORLY_WORDED",
      "OFF_TOPIC",
      "TOO_EASY",
      "TOO_HARD",
      "DUPLICATE",
      "OTHER",
    ];

    if (!VALID_REASONS.includes(reason)) {
      throw {
        code: "INVALID_REJECTION_REASON",
        message: `Rejection reason must be one of: 
                  ${VALID_REASONS.join(", ")}`,
        statusCode: 400,
      };
    }

    const quiz = await this.findAndVerifyQuiz(quizId, rejectedBy);

    await prisma.quiz.update({
      where: { id: quizId },
      data: {
        generationStatus: "REJECTED",
        rejectionReason: reason,
      },
    });

    const thresholdStatus = await this.getThresholdStatus(
      quiz.contentId,
      quiz.quizType
    );

    return {
      quiz_id: quizId,
      generation_status: "REJECTED",
      rejection_reason: reason,
      content_threshold_status: thresholdStatus,
    };
  }

  // -------------------------------------------------------
  // EDIT QUIZ
  // -------------------------------------------------------
  async editQuiz(
    quizId: string,
    editorId: string,
    data: {
      question_text?: string;
      options?: string[];
      correct_option_index?: number;
      cognitive_level?: string;
      difficulty?: string;
      explanation?: string;
    }
  ) {
    const quiz = await this.findAndVerifyQuiz(quizId, editorId);

    const updateData: any = {
      wasEdited: true,
      updatedAt: new Date(),
    };

    // Preserve originals on first edit only
    if (!quiz.wasEdited) {
      if (data.question_text !== undefined) {
        updateData.originalQuestionText = quiz.questionText;
      }
      if (data.options !== undefined) {
        updateData.originalOptions = quiz.options;
      }
    }

    if (data.question_text !== undefined) {
      updateData.questionText = data.question_text;
    }

    if (data.options !== undefined) {
      if (data.options.length !== 4) {
        throw {
          code: "VALIDATION_ERROR",
          message: "Exactly 4 options are required",
          statusCode: 400,
        };
      }
      updateData.options = data.options;
    }

    if (data.correct_option_index !== undefined) {
      if (
        data.correct_option_index < 0 ||
        data.correct_option_index > 3
      ) {
        throw {
          code: "VALIDATION_ERROR",
          message: "correct_option_index must be 0, 1, 2, or 3",
          statusCode: 400,
        };
      }
      updateData.correctOptionIndex = data.correct_option_index;
    }

    if (data.cognitive_level) updateData.cognitiveLevel = data.cognitive_level;
    if (data.difficulty) updateData.difficulty = data.difficulty;
    if (data.explanation !== undefined) {
      updateData.explanation = data.explanation;
    }

    const updated = await prisma.quiz.update({
      where: { id: quizId },
      data: updateData,
    });

    return {
      quiz_id: updated.id,
      question_text: updated.questionText,
      options: updated.options,
      correct_option_index: updated.correctOptionIndex,
      cognitive_level: updated.cognitiveLevel,
      difficulty: updated.difficulty,
      explanation: updated.explanation,
      was_edited: updated.wasEdited,
      generation_status: updated.generationStatus,
      updated_at: updated.updatedAt,
    };
  }

  // -------------------------------------------------------
  // CREATE MANUAL QUIZ
  // -------------------------------------------------------
  async createManualQuiz(
    contentId: string,
    createdBy: string,
    data: {
      quiz_type: string;
      question_text: string;
      options: string[];
      correct_option_index: number;
      cognitive_level: string;
      difficulty: string;
      explanation?: string;
    }
  ) {
    const content = await prisma.content.findUnique({
      where: { id: contentId, deletedAt: null },
      select: { batchId: true },
    });

    if (!content) {
      throw { code: "CONTENT_NOT_FOUND", statusCode: 404 };
    }

    const quiz = await prisma.quiz.create({
      data: {
        contentId,
        batchId: content.batchId,
        quizType: data.quiz_type as any,
        questionText: data.question_text,
        options: data.options,
        correctOptionIndex: data.correct_option_index,
        cognitiveLevel: data.cognitive_level as any,
        difficulty: data.difficulty as any,
        explanation: data.explanation ?? null,
        isAiGenerated: false,
        generationStatus: "APPROVED",
        approvedBy: createdBy,
        approvedAt: new Date(),
        availableFrom: new Date(),
      },
    });

    const thresholdStatus = await this.getThresholdStatus(
      contentId,
      data.quiz_type
    );

    return {
      quiz_id: quiz.id,
      generation_status: "APPROVED",
      is_ai_generated: false,
      content_threshold_status: thresholdStatus,
    };
  }

  // -------------------------------------------------------
  // REVOKE APPROVAL
  // -------------------------------------------------------
  async revokeApproval(quizId: string, revokedBy: string) {
    const quiz = await this.findAndVerifyQuiz(quizId, revokedBy);

    if (quiz.generationStatus !== "APPROVED") {
      throw {
        code: "NOT_APPROVED",
        message: "Only approved questions can have approval revoked",
        statusCode: 400,
      };
    }

    // Check if students have answered this question
    const responseCount = await prisma.quizResponse.count({
      where: { quizId },
    });

    if (responseCount > 0) {
      throw {
        code: "CANNOT_REVOKE_ANSWERED_QUESTION",
        message: `This question has been answered by ${responseCount} 
                  student${responseCount !== 1 ? "s" : ""} and cannot 
                  be revoked.`,
        statusCode: 400,
        details: { response_count: responseCount },
      };
    }

    await prisma.quiz.update({
      where: { id: quizId },
      data: {
        generationStatus: "PENDING_REVIEW",
        approvedBy: null,
        approvedAt: null,
      },
    });

    return {
      quiz_id: quizId,
      generation_status: "PENDING_REVIEW",
      approved_by: null,
      approved_at: null,
    };
  }

  // -------------------------------------------------------
  // GET REVIEW STATS
  // -------------------------------------------------------
  async getReviewStats(batchId: string) {
    const [allQuizzes, byContent, byCognitive] = await Promise.all([
      prisma.quiz.groupBy({
        by: ["generationStatus", "wasEdited"],
        where: { batchId },
        _count: { id: true },
      }),
      prisma.quiz.groupBy({
        by: ["contentId", "generationStatus", "wasEdited"],
        where: { batchId },
        _count: { id: true },
      }),
      prisma.quiz.groupBy({
        by: ["cognitiveLevel", "generationStatus"],
        where: { batchId },
        _count: { id: true },
      }),
    ]);

    // Process rejection reasons
    const rejectedQuizzes = await prisma.quiz.findMany({
      where: { batchId, generationStatus: "REJECTED" },
      select: { rejectionReason: true },
    });

    const rejectionBreakdown: Record<string, number> = {};
    for (const quiz of rejectedQuizzes) {
      const reason = quiz.rejectionReason ?? "OTHER";
      rejectionBreakdown[reason] =
        (rejectionBreakdown[reason] ?? 0) + 1;
    }

    return {
      overall: this.calculateOverallStats(allQuizzes),
      rejection_breakdown: rejectionBreakdown,
      by_cognitive_level: this.calculateCognitiveLevelStats(
        byCognitive
      ),
    };
  }

  // -------------------------------------------------------
  // PRIVATE HELPERS
  // -------------------------------------------------------

  private async findAndVerifyQuiz(quizId: string, userId: string) {
    const quiz = await prisma.quiz.findUnique({
      where: { id: quizId },
      include: {
        content: {
          include: {
            batch: {
              include: {
                mentors: { select: { mentorId: true } },
              },
            },
          },
        },
      },
    });

    if (!quiz) {
      throw { code: "QUIZ_NOT_FOUND", statusCode: 404 };
    }

    // Verify user has access (is mentor for this batch or admin)
    const user = await prisma.user.findUnique({
      where: { id: userId },
      select: { role: true },
    });

    if (user?.role === "MENTOR") {
      const isAssigned = quiz.content.batch.mentors.some(
        (bm) => bm.mentorId === userId
      );
      if (!isAssigned) {
        throw { code: "PERMISSION_DENIED", statusCode: 403 };
      }
    }

    return quiz;
  }

  private async getThresholdStatus(
    contentId: string,
    quizType: string
  ) {
    const minimum =
      quizType === "QUICK_ASSESSMENT"
        ? QUICK_ASSESSMENT_MINIMUM
        : RETENTION_MINIMUM;

    const approvedCount = await prisma.quiz.count({
      where: {
        contentId,
        quizType: quizType as any,
        generationStatus: "APPROVED",
      },
    });

    return {
      quiz_type: quizType,
      approved_count: approvedCount,
      minimum_required: minimum,
      threshold_met: approvedCount >= minimum,
    };
  }

  private async getContentGroupSummaries(batchId: string) {
    const grouped = await prisma.quiz.groupBy({
      by: ["contentId", "quizType", "generationStatus"],
      where: { batchId },
      _count: { id: true },
    });

    const contentIds = [...new Set(grouped.map((g) => g.contentId))];
    const contentDetails = await prisma.content.findMany({
      where: { id: { in: contentIds } },
      select: { id: true, title: true, isPublished: true },
    });

    const contentMap = Object.fromEntries(
      contentDetails.map((c) => [c.id, c])
    );

    const groupMap: Record<string, any> = {};

    for (const item of grouped) {
      if (!groupMap[item.contentId]) {
        const content = contentMap[item.contentId];
        groupMap[item.contentId] = {
          content_id: item.contentId,
          content_title: content?.title ?? "Unknown",
          content_published: content?.isPublished ?? false,
          pending_count: 0,
          approved_count: 0,
          rejected_count: 0,
          quick_assessment: {
            pending: 0,
            approved: 0,
            minimum_required: QUICK_ASSESSMENT_MINIMUM,
            threshold_met: false,
          },
          retention: {
            pending: 0,
            approved: 0,
            minimum_required: RETENTION_MINIMUM,
            threshold_met: false,
          },
        };
      }

      const group = groupMap[item.contentId];
      const count = item._count.id;
      const typeKey =
        item.quizType === "QUICK_ASSESSMENT"
          ? "quick_assessment"
          : "retention";
      const statusKey =
        item.generationStatus.toLowerCase() as
          | "pending_review"
          | "approved"
          | "rejected";

      if (statusKey === "pending_review") {
        group.pending_count += count;
        group[typeKey].pending += count;
      } else if (statusKey === "approved") {
        group.approved_count += count;
        group[typeKey].approved += count;
      } else if (statusKey === "rejected") {
        group.rejected_count += count;
      }
    }

    // Calculate threshold met status
    for (const group of Object.values(groupMap)) {
      group.quick_assessment.threshold_met =
        group.quick_assessment.approved >=
        QUICK_ASSESSMENT_MINIMUM;
      group.retention.threshold_met =
        group.retention.approved >= RETENTION_MINIMUM;
    }

    return Object.values(groupMap);
  }

  private async verifyMentorBatchAccess(
    batchId: string,
    userId: string
  ) {
    const user = await prisma.user.findUnique({
      where: { id: userId },
      select: { role: true },
    });

    if (
      user?.role === "ADMIN" ||
      user?.role === "SUPER_ADMIN"
    ) {
      return;
    }

    const assignment = await prisma.batchMentor.findFirst({
      where: { batchId, mentorId: userId },
    });

    if (!assignment) {
      throw { code: "PERMISSION_DENIED", statusCode: 403 };
    }
  }

  private calculateOverallStats(groupedData: any[]) {
    let total = 0;
    let approvedWithoutEdit = 0;
    let approvedWithEdit = 0;
    let rejected = 0;

    for (const item of groupedData) {
      total += item._count.id;
      if (item.generationStatus === "APPROVED" && !item.wasEdited) {
        approvedWithoutEdit += item._count.id;
      } else if (
        item.generationStatus === "APPROVED" &&
        item.wasEdited
      ) {
        approvedWithEdit += item._count.id;
      } else if (item.generationStatus === "REJECTED") {
        rejected += item._count.id;
      }
    }

    return {
      total_generated: total,
      total_approved_without_edit: approvedWithoutEdit,
      total_approved_with_edit: approvedWithEdit,
      total_rejected: rejected,
      approval_rate_without_edit:
        total > 0
          ? Math.round((approvedWithoutEdit / total) * 1000) / 10
          : 0,
      overall_approval_rate:
        total > 0
          ? Math.round(
              ((approvedWithoutEdit + approvedWithEdit) / total) * 1000
            ) / 10
          : 0,
    };
  }

  private calculateCognitiveLevelStats(groupedData: any[]) {
    const levels: Record<string, any> = {};

    for (const item of groupedData) {
      const level = item.cognitiveLevel;
      if (!levels[level]) {
        levels[level] = { generated: 0, approved: 0, rejected: 0 };
      }
      levels[level].generated += item._count.id;
      if (item.generationStatus === "APPROVED") {
        levels[level].approved += item._count.id;
      } else if (item.generationStatus === "REJECTED") {
        levels[level].rejected += item._count.id;
      }
    }

    for (const level of Object.values(levels)) {
      level.approval_rate =
        level.generated > 0
          ? Math.round((level.approved / level.generated) * 1000) / 10
          : 0;
    }

    return levels;
  }

  private chunkArray<T>(array: T[], size: number): T[][] {
    const chunks: T[][] = [];
    for (let i = 0; i < array.length; i += size) {
      chunks.push(array.slice(i, i + size));
    }
    return chunks;
  }
}
```

## 8.1 Error Code Reference

```
QUIZ_NOT_FOUND               : 404 — Quiz ID does not exist
ALREADY_APPROVED             : 400 — Cannot approve already approved
NOT_APPROVED                 : 400 — Cannot revoke non-approved quiz
CANNOT_REVOKE_ANSWERED_QUESTION: 400 — Students answered this question
INVALID_REJECTION_REASON     : 400 — Reason not in allowed list
PERMISSION_DENIED            : 403 — Mentor not assigned to batch
BATCH_NOT_FOUND              : 404 — Batch does not exist
CONTENT_NOT_FOUND            : 404 — Content does not exist
VALIDATION_ERROR             : 400 — Input validation failure
```

## 8.2 Concurrent Approval Handling

Two mentors could theoretically try to approve or reject the same
question simultaneously (rare in Phase One with small teams but
possible). Handle this with Prisma's update where clause:

```typescript
// When approving, only update if still PENDING_REVIEW
const updated = await prisma.quiz.updateMany({
  where: {
    id: quizId,
    generationStatus: "PENDING_REVIEW", // Guard condition
  },
  data: { generationStatus: "APPROVED" ... },
});

if (updated.count === 0) {
  // Question was already acted on by someone else
  throw {
    code: "ALREADY_APPROVED",
    message: "This question was already acted on",
    statusCode: 409,
  };
}
```

# 10. Code Examples

## 10.1 How Feature 07 Checks Quiz Availability Before Serving to Students

Feature 07 (Quiz Taking System) queries only approved quizzes past
their available_from date. The threshold check ensures students
never see a quiz with too few questions:

```typescript
// Used by Feature 07 before serving quiz to student
const isQuizReadyForStudents = async (
  contentId: string,
  quizType: string
): Promise<boolean> => {
  const MINIMUM = quizType === "QUICK_ASSESSMENT" ? 5 : 8;
  const now = new Date();

  const approvedCount = await prisma.quiz.count({
    where: {
      contentId,
      quizType: quizType as any,
      generationStatus: "APPROVED",
      availableFrom: { lte: now },
    },
  });

  return approvedCount >= MINIMUM;
};
```

# 11. Performance Optimization

## 11.1 Review Queue Query Optimization

The review queue fetches all PENDING_REVIEW questions for a batch.
With the partial index on (batch_id, generation_status, created_at)
WHERE generation_status = 'PENDING_REVIEW', this query is fast even
for batches with hundreds of content items.

As questions are approved or rejected, they leave the partial index
automatically, keeping it compact. The index size grows only when
new content is uploaded and new questions are generated.

## 11.2 Optimistic UI Updates

All approve, reject, and edit actions use optimistic UI updates in
the useReviewQueue hook — questions are removed from or updated in
the local state immediately when the action is triggered, before
the API response arrives. If the API call fails, the state is
rolled back. This makes the review workflow feel instant even on
slow network connections.

```typescript
// Optimistic update pattern
const approveQuestion = useCallback(
  async (quizId: string) => {
    // Optimistically remove from queue immediately
    removeQuestion(quizId);

    try {
      const result = await api.post(`/api/quizzes/${quizId}/approve`);
      return result.data.data.content_threshold_status;
    } catch (error) {
      // Roll back — add question back to queue
      setState((prev) => ({
        ...prev,
        questions: [...prev.questions, originalQuestion],
      }));
      throw error;
    }
  },
  [removeQuestion]
);
```

## 11.3 Batch Approve Performance

The batch approve endpoint processes IDs in chunks of 10 to avoid
overwhelming the database with a single massive transaction.
For Phase One with batches of up to 500 students and 30–50 questions
per content item, a batch of 50 approvals takes approximately
200–400ms server-side.
