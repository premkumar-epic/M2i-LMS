# 1. Feature Overview

## 1.1 What Is This Feature?

The Quiz Taking System is the student-facing assessment interface
of M2i_LMS. It is the feature that transforms approved quiz questions
— generated by Feature 04 and reviewed by Feature 05 — into a
structured, measured assessment experience for students. Every quiz
response a student submits feeds directly into the metrics engine
that powers the nine learning dimension scores.

This feature handles the complete assessment lifecycle from the
student's perspective: discovering available quizzes, taking them
through a structured interface, submitting answers, receiving
immediate scored feedback, and reviewing their quiz history over
time. It also handles the backend logic of recording responses,
preventing duplicate submissions, auto-grading, and triggering
metric recalculations.

## 1.2 Why This Feature Exists

Without a quiz taking system, the platform has no mechanism to
measure what students actually understood from the content. Video
views and attendance data tell us that a student was present. Quiz
responses tell us whether they comprehended, can apply, and retain
what was taught. This is the distinction between engagement data and
learning data — and the quiz system is what produces learning data.

The quiz system is also the primary source of data for six of the
nine learning dimensions: Learning Velocity, Knowledge Retention,
Problem-Solving Approach, Conceptual Depth, Error Recovery and
Resilience, and Communication and Articulation. Without reliable
quiz data, these dimensions cannot be calculated.

## 1.3 Key Design Decisions

**Answer option randomization per student:** The correct answer is
never always in the same position. Each student sees options in a
different order. This prevents copying and makes each student's
response genuinely theirs. The canonical correct_option_index is
preserved in the database — only the display order changes.

**Single atomic submission:** All answers for a quiz session are
submitted in one API call, not one call per question. This prevents
partial submissions where some answers are recorded and others are
not due to network failure.

**Auto-save to localStorage:** Answer selections are saved to
localStorage every time the student changes an answer. If the browser
crashes during a quiz, the student can resume from where they left
off. The auto-save expires after 24 hours.

**No retakes by default:** Once a quiz is submitted, it cannot be
retaken. This ensures that the first honest attempt is what gets
recorded. Mentors can unlock retakes for specific quizzes if needed.

**Immediate feedback:** Results are shown immediately after
submission — score, question-level breakdown, correct answers, and
explanations. Delayed feedback reduces the learning value of
assessments.

**Idempotent submission endpoint:** The backend is designed to handle
duplicate submissions gracefully. If the same student submits the
same quiz twice (due to network retry), only the first submission is
recorded and the second returns the existing results.

# 3. Data Model

## 3.1 QuizResponses Table

Stores every individual question response from every student.
This is the most granular assessment data in the system and the
primary input for the metrics engine.

```sql
CREATE TABLE quiz_responses (
  id                      UUID        PRIMARY KEY
                                      DEFAULT gen_random_uuid(),
  student_id              UUID        NOT NULL
                                      REFERENCES users(id),
  quiz_id                 UUID        NOT NULL
                                      REFERENCES quizzes(id),
  content_id              UUID        NOT NULL
                                      REFERENCES content(id),
  batch_id                UUID        NOT NULL
                                      REFERENCES batches(id),
  attempt_id              UUID        NOT NULL,
  selected_option_index   INTEGER     NOT NULL
                                      CHECK (selected_option_index
                                        BETWEEN 0 AND 3),
  is_correct              BOOLEAN     NOT NULL,
  time_to_answer_seconds  INTEGER     DEFAULT NULL,
  submitted_at            TIMESTAMP   NOT NULL DEFAULT NOW()
);

-- One response per student per quiz question per attempt
CREATE UNIQUE INDEX idx_quiz_responses_student_quiz_attempt
  ON quiz_responses(student_id, quiz_id, attempt_id);

-- Indexes for analytics queries
CREATE INDEX idx_quiz_responses_student_id
  ON quiz_responses(student_id);

CREATE INDEX idx_quiz_responses_quiz_id
  ON quiz_responses(quiz_id);

CREATE INDEX idx_quiz_responses_content_id
  ON quiz_responses(content_id);

CREATE INDEX idx_quiz_responses_batch_id
  ON quiz_responses(batch_id);

CREATE INDEX idx_quiz_responses_attempt_id
  ON quiz_responses(attempt_id);

CREATE INDEX idx_quiz_responses_submitted_at
  ON quiz_responses(student_id, submitted_at DESC);
```

### Column Definitions

**id:** UUID primary key.

**student_id:** The student who submitted this response.

**quiz_id:** The specific quiz question being answered.
Foreign key to quizzes table.

**content_id:** Denormalized for query efficiency — allows
fetching all responses for a content item without joining
through quizzes.

**batch_id:** Denormalized for batch-level analytics.

**attempt_id:** UUID that groups all responses from a single
quiz session together. All questions answered in one sitting
share the same attempt_id. This allows the system to identify
retakes (different attempt_id for the same student and content).

**selected_option_index:** The canonical index (0-3) of the
option the student chose. Note: this is the canonical index,
not the display index. The frontend must map the display
order back to canonical before submitting.

**is_correct:** Pre-calculated at submission time by comparing
selected_option_index to the quiz's correct_option_index.
Stored for query efficiency — the metrics engine reads this
field directly rather than re-calculating.

**time_to_answer_seconds:** How long the student spent on this
question from when it was first displayed to when they selected
an answer. NULL if timing was not tracked (e.g., resumed from
localStorage). Used for the Problem-Solving Approach dimension.

**submitted_at:** When the response was submitted.

## 3.2 QuizAttempts Table

Stores a summary record for each complete quiz session —
the overall score and metadata. Makes it efficient to display
quiz history without querying all individual responses.

```sql
CREATE TABLE quiz_attempts (
  id                    UUID          PRIMARY KEY
                                      DEFAULT gen_random_uuid(),
  attempt_id            UUID          NOT NULL UNIQUE,
  student_id            UUID          NOT NULL
                                      REFERENCES users(id),
  content_id            UUID          NOT NULL
                                      REFERENCES content(id),
  batch_id              UUID          NOT NULL
                                      REFERENCES batches(id),
  quiz_type             VARCHAR(20)   NOT NULL
                                      CHECK (quiz_type IN (
                                        'QUICK_ASSESSMENT',
                                        'RETENTION'
                                      )),
  total_questions       INTEGER       NOT NULL,
  correct_answers       INTEGER       NOT NULL,
  score_percentage      DECIMAL(5,2)  NOT NULL,
  time_taken_seconds    INTEGER       DEFAULT NULL,
  started_at            TIMESTAMP     DEFAULT NULL,
  completed_at          TIMESTAMP     NOT NULL DEFAULT NOW()
);

-- One attempt per student per content per quiz type
-- (Phase One: no retakes, so this is effectively unique)
CREATE INDEX idx_quiz_attempts_student_content
  ON quiz_attempts(student_id, content_id, quiz_type);

CREATE INDEX idx_quiz_attempts_student_id
  ON quiz_attempts(student_id);

CREATE INDEX idx_quiz_attempts_batch_id
  ON quiz_attempts(batch_id);

CREATE INDEX idx_quiz_attempts_completed_at
  ON quiz_attempts(student_id, completed_at DESC);
```

### Column Definitions

**attempt_id:** Shared UUID between quiz_attempts and
quiz_responses — links the summary to the detail records.

**total_questions:** How many questions were in this attempt.

**correct_answers:** How many the student answered correctly.

**score_percentage:** (correct_answers / total_questions) * 100,
stored for fast retrieval without recalculation.

**time_taken_seconds:** Total time from first question display
to submission. Calculated from started_at to completed_at.

**started_at:** When the student opened the first question.
Set by the frontend when the quiz begins.

## 3.3 Prisma Schema

```prisma
model QuizResponse {
  id                    String    @id
                                  @default(dbgenerated(
                                    "gen_random_uuid()"))
                                  @db.Uuid
  studentId             String    @map("student_id") @db.Uuid
  quizId                String    @map("quiz_id") @db.Uuid
  contentId             String    @map("content_id") @db.Uuid
  batchId               String    @map("batch_id") @db.Uuid
  attemptId             String    @map("attempt_id") @db.Uuid
  selectedOptionIndex   Int       @map("selected_option_index")
  isCorrect             Boolean   @map("is_correct")
  timeToAnswerSeconds   Int?      @map("time_to_answer_seconds")
  submittedAt           DateTime  @default(now())
                                  @map("submitted_at")

  student   User    @relation(fields: [studentId],
                              references: [id])
  quiz      Quiz    @relation(fields: [quizId],
                              references: [id])
  content   Content @relation(fields: [contentId],
                              references: [id])

  @@unique([studentId, quizId, attemptId])
  @@map("quiz_responses")
}

model QuizAttempt {
  id                String      @id
                                @default(dbgenerated(
                                  "gen_random_uuid()"))
                                @db.Uuid
  attemptId         String      @unique @map("attempt_id")
                                @db.Uuid
  studentId         String      @map("student_id") @db.Uuid
  contentId         String      @map("content_id") @db.Uuid
  batchId           String      @map("batch_id") @db.Uuid
  quizType          QuizType    @map("quiz_type")
  totalQuestions    Int         @map("total_questions")
  correctAnswers    Int         @map("correct_answers")
  scorePercentage   Decimal     @map("score_percentage")
                                @db.Decimal(5, 2)
  timeTakenSeconds  Int?        @map("time_taken_seconds")
  startedAt         DateTime?   @map("started_at")
  completedAt       DateTime    @default(now())
                                @map("completed_at")

  student   User    @relation(fields: [studentId],
                              references: [id])
  content   Content @relation(fields: [contentId],
                              references: [id])

  @@map("quiz_attempts")
}
```

```json
          "To allow Node.js to handle multiple I/O operations

          "To compile JavaScript code to machine code

          "To synchronize data between multiple

        "question_number": 1
```

**Important notes on display_order:** The options array contains

canonical index in the database. For example, display_order: [1, 0, 2, 3]
means display position 0 shows canonical option 1, display position 1
shows canonical option 0, etc. When submitting, the student sends
their selected display index, and the backend uses display_order to
convert it to the canonical index for correctness checking.

## 4.2 Quiz Submission Endpoints

### POST /api/quizzes/submit

**Access:** STUDENT (enrolled in batch)

**Purpose:** Submit all quiz responses in a single atomic operation.

**Request Body:**

```json
{
  "content_id": "content-uuid-1",
  "quiz_type": "QUICK_ASSESSMENT",
  "attempt_id": "attempt-uuid-new",
  "started_at": "2026-04-02T11:00:00Z",
  "responses": [
    {
      "quiz_id": "quiz-uuid-1",
      "selected_display_index": 1,
      "time_to_answer_seconds": 45
    },
    {
      "quiz_id": "quiz-uuid-2",
      "selected_display_index": 0,
      "time_to_answer_seconds": 23
    }
  ],
  "display_orders": {
    "quiz-uuid-1": [1, 0, 2, 3],
    "quiz-uuid-2": [2, 3, 0, 1]
  }
}
```

**Field Explanations:**

**attempt_id:** UUID generated by the frontend at the start of
the quiz session. Used for idempotency — if the same attempt_id
is submitted twice, the second submission returns the first
result without creating duplicate records.

**started_at:** When the student opened the first question.
Used to calculate total_time_taken_seconds.

**selected_display_index:** The display position (0-3) the student
selected. The backend uses display_orders to convert this to the
canonical index.

**display_orders:** A map of quiz_id to display_order array,
sent back from the /available endpoint. The backend uses this to
convert display indices to canonical indices for correctness
checking.

**Validation Rules:**

```
content_id        : required, UUID
quiz_type         : required, QUICK_ASSESSMENT or RETENTION
attempt_id        : required, UUID
started_at        : required, ISO 8601 datetime
responses         : required, array, must contain one entry
                    for every question returned by /available
display_orders    : required, map of quiz_id to array of 4 integers
responses[].quiz_id               : required, UUID
responses[].selected_display_index: required, integer 0-3
responses[].time_to_answer_seconds: optional, integer
```

**Success Response (200 OK):**

```json
{
  "success": true,
  "data": {
```

## 5.2 useQuizSession Hook

This is the central state manager for the entire quiz taking
experience. It manages question navigation, answer selection,
auto-save, submission, and results.

```typescript
// hooks/useQuizSession.ts
"use client";

import { useState, useCallback, useEffect } from "react";
import { v4 as uuidv4 } from "uuid";
import api from "@/lib/api";

export type QuizQuestion = {
  quiz_id: string;
  question_text: string;
  options: string[];
  display_order: number[];
  question_number: number;
};

export type QuizSessionState =
  | "intro"
  | "taking"
  | "reviewing"
  | "submitting"
  | "results"
  | "error";

export type QuizResult = {
  attempt_id: string;
  score: {
    correct_answers: number;
    total_questions: number;
    score_percentage: number;
    time_taken_seconds: number;
  };
  results: Array<{
    quiz_id: string;
    question_text: string;
    your_answer: string;
    correct_answer: string;
    is_correct: boolean;
    explanation: string | null;
  }>;
};

type AutoSaveState = {
  content_id: string;
  quiz_type: string;
  attempt_id: string;
  started_at: string;
  answers: Record<string, number>; // quiz_id → selected_display_index
  time_per_question: Record<string, number>; // quiz_id → seconds
};

const AUTO_SAVE_KEY = (contentId: string, quizType: string) =>
  `quiz_session_${contentId}_${quizType}`;

const AUTO_SAVE_EXPIRY_HOURS = 24;

export function useQuizSession(
  contentId: string,
  quizType: string
) {
  const [sessionState, setSessionState] =
    useState<QuizSessionState>("intro");
  const [questions, setQuestions] = useState<QuizQuestion[]>([]);
  const [currentQuestionIndex, setCurrentQuestionIndex] = useState(0);
  const [answers, setAnswers] = useState<Record<string, number>>({});
  const [timePerQuestion, setTimePerQuestion] = useState
    Record<string, number>
  >({});
  const [attemptId] = useState(() => uuidv4());
  const [startedAt, setStartedAt] = useState<string | null>(null);
  const [questionStartTime, setQuestionStartTime] = useState<number>(
    Date.now()
  );
  const [result, setResult] = useState<QuizResult | null>(null);
  const [error, setError] = useState<string | null>(null);
  const [isLoading, setIsLoading] = useState(false);

  // -------------------------------------------------------
  // Auto-save to localStorage
  // -------------------------------------------------------
  const saveToLocalStorage = useCallback(
    (updatedAnswers: Record<string, number>) => {
      const state: AutoSaveState = {
        content_id: contentId,
        quiz_type: quizType,
        attempt_id: attemptId,
        started_at: startedAt ?? new Date().toISOString(),
        answers: updatedAnswers,
        time_per_question: timePerQuestion,
      };

      const saveData = {
        state,
        saved_at: Date.now(),
        expires_at:
          Date.now() + AUTO_SAVE_EXPIRY_HOURS * 60 * 60 * 1000,
      };

      localStorage.setItem(
        AUTO_SAVE_KEY(contentId, quizType),
        JSON.stringify(saveData)
      );
    },
    [contentId, quizType, attemptId, startedAt, timePerQuestion]
  );

  const loadFromLocalStorage = useCallback((): AutoSaveState | null => {
    try {
      const raw = localStorage.getItem(
        AUTO_SAVE_KEY(contentId, quizType)
      );
      if (!raw) return null;

      const saveData = JSON.parse(raw);

      // Check expiry
      if (Date.now() > saveData.expires_at) {
        localStorage.removeItem(AUTO_SAVE_KEY(contentId, quizType));
        return null;
      }

      return saveData.state;
    } catch {
      return null;
    }
  }, [contentId, quizType]);

  const clearAutoSave = useCallback(() => {
    localStorage.removeItem(AUTO_SAVE_KEY(contentId, quizType));
  }, [contentId, quizType]);

  // -------------------------------------------------------
  // Start quiz — fetch questions
  // -------------------------------------------------------
  const startQuiz = useCallback(async () => {
    setIsLoading(true);
    setError(null);

    try {
      // Check for saved state
      const savedState = loadFromLocalStorage();

      const response = await api.get(
        `/api/content/${contentId}/quizzes/available`,
        { params: { quiz_type: quizType } }
      );

      const { questions: fetchedQuestions } = response.data.data;
      setQuestions(fetchedQuestions);

      const now = new Date().toISOString();
      setStartedAt(now);
      setQuestionStartTime(Date.now());

      // Restore saved answers if available
      if (savedState && Object.keys(savedState.answers).length > 0) {
        setAnswers(savedState.answers);
        setTimePerQuestion(savedState.time_per_question);
      }

      setSessionState("taking");
    } catch (err: any) {
      const code = err.response?.data?.error?.code;
      if (code === "QUIZ_ALREADY_SUBMITTED") {
        setError("You have already completed this quiz.");
      } else if (code === "NO_QUIZZES_AVAILABLE") {
        setError("No quiz questions are available yet for this content.");
      } else {
        setError("Failed to load quiz. Please try again.");
      }
      setSessionState("error");
    } finally {
      setIsLoading(false);
    }
  }, [contentId, quizType, loadFromLocalStorage]);

  // -------------------------------------------------------
  // Select answer
  // -------------------------------------------------------
  const selectAnswer = useCallback(
    (quizId: string, displayIndex: number) => {
      // Record time spent on this question
      const elapsed = Math.floor(
        (Date.now() - questionStartTime) / 1000
      );

      setTimePerQuestion((prev) => ({
        ...prev,
        [quizId]: elapsed,
      }));

      const updatedAnswers = { ...answers, [quizId]: displayIndex };
      setAnswers(updatedAnswers);
      saveToLocalStorage(updatedAnswers);
    },
    [answers, questionStartTime, saveToLocalStorage]
  );

  // -------------------------------------------------------
  // Navigate questions
  // -------------------------------------------------------
  const goToQuestion = useCallback(
    (index: number) => {
      if (index < 0 || index >= questions.length) return;
      setCurrentQuestionIndex(index);
      setQuestionStartTime(Date.now());
    },
    [questions.length]
  );

  const goToNext = useCallback(() => {
    goToQuestion(currentQuestionIndex + 1);
  }, [currentQuestionIndex, goToQuestion]);

  const goToPrevious = useCallback(() => {
    goToQuestion(currentQuestionIndex - 1);
  }, [currentQuestionIndex, goToQuestion]);

  const goToReview = useCallback(() => {
    setSessionState("reviewing");
  }, []);

  const goBackToQuiz = useCallback(() => {
    setSessionState("taking");
  }, []);

  // -------------------------------------------------------
  // Submit quiz
  // -------------------------------------------------------
  const submitQuiz = useCallback(async () => {
    if (sessionState === "submitting") return;

    setSessionState("submitting");
    setError(null);

    try {
      // Build display_orders map from questions
      const displayOrders: Record<string, number[]> = {};
      for (const q of questions) {
        displayOrders[q.quiz_id] = q.display_order;
      }

      // Build responses — map display index to canonical index
      const responses = questions.map((q) => {
        const selectedDisplayIndex = answers[q.quiz_id] ?? 0;

        return {
          quiz_id: q.quiz_id,
          selected_display_index: selectedDisplayIndex,
          time_to_answer_seconds: timePerQuestion[q.quiz_id] ?? null,
        };
      });

      const response = await api.post("/api/quizzes/submit", {
        content_id: contentId,
        quiz_type: quizType,
        attempt_id: attemptId,
        started_at: startedAt,
        responses,
        display_orders: displayOrders,
      });

      clearAutoSave();
      setResult(response.data.data);
      setSessionState("results");

    } catch (err: any) {
      const code = err.response?.data?.error?.code;
      if (code === "QUIZ_ALREADY_SUBMITTED") {
        setError("This quiz was already submitted.");
        clearAutoSave();
        setSessionState("error");
      } else {
        setError("Submission failed. Please try again.");
        setSessionState("reviewing");
      }
    }
  }, [
    sessionState,
    questions,
    answers,
    timePerQuestion,
    contentId,
    quizType,
    attemptId,
    startedAt,
    clearAutoSave,
  ]);

  const answeredCount = Object.keys(answers).length;
  const totalQuestions = questions.length;
  const currentQuestion = questions[currentQuestionIndex] ?? null;
  const isAllAnswered = answeredCount === totalQuestions;

  return {
    sessionState,
    questions,
    currentQuestion,
    currentQuestionIndex,
    answers,
    result,
    error,
    isLoading,
    isAllAnswered,
    answeredCount,
    totalQuestions,
    startQuiz,
    selectAnswer,
    goToQuestion,
    goToNext,
    goToPrevious,
    goToReview,
    goBackToQuiz,
    submitQuiz,
  };
}
```

```typescript
// services/quizTaking.service.ts
import { PrismaClient } from "@prisma/client";
import { v4 as uuidv4 } from "uuid";
import { metricsQueue } from "../queues/metrics.queue";

const prisma = new PrismaClient();

const QUICK_ASSESSMENT_MINIMUM = 5;
const RETENTION_MINIMUM = 8;

export class QuizTakingService {

  // -------------------------------------------------------
  // GET QUIZ STATUS FOR CONTENT ITEM
  // -------------------------------------------------------
  async getQuizStatus(contentId: string, studentId: string) {
    const content = await prisma.content.findUnique({
      where: { id: contentId, deletedAt: null },
      select: { title: true, batchId: true },
    });

    if (!content) {
      throw { code: "CONTENT_NOT_FOUND", statusCode: 404 };
    }

    // Verify student enrollment
    await this.verifyStudentEnrollment(studentId, content.batchId);

    const now = new Date();

    const [quickCount, retentionCount,
           quickAttempt, retentionAttempt] = await Promise.all([
      // Count approved Quick Assessment questions
      prisma.quiz.count({
        where: {
          contentId,
          quizType: "QUICK_ASSESSMENT",
          generationStatus: "APPROVED",
          availableFrom: { lte: now },
        },
      }),
      // Count approved Retention questions
      prisma.quiz.count({
        where: {
          contentId,
          quizType: "RETENTION",
          generationStatus: "APPROVED",
          availableFrom: { lte: now },
        },
      }),
      // Student's Quick Assessment attempt
      prisma.quizAttempt.findFirst({
        where: {
          studentId,
          contentId,
          quizType: "QUICK_ASSESSMENT",
        },
        orderBy: { completedAt: "desc" },
      }),
      // Student's Retention attempt
      prisma.quizAttempt.findFirst({
        where: {
          studentId,
          contentId,
          quizType: "RETENTION",
        },
        orderBy: { completedAt: "desc" },
      }),
    ]);

    // Get availability dates
    const quickAvailableFrom = await prisma.quiz.findFirst({
      where: {
        contentId,
        quizType: "QUICK_ASSESSMENT",
        generationStatus: "APPROVED",
      },
      select: { availableFrom: true },
      orderBy: { availableFrom: "asc" },
    });

    const retentionAvailableFrom = await prisma.quiz.findFirst({
      where: {
        contentId,
        quizType: "RETENTION",
        generationStatus: "APPROVED",
      },
      select: { availableFrom: true },
      orderBy: { availableFrom: "asc" },
    });

    const quickAvailable =
      quickCount >= QUICK_ASSESSMENT_MINIMUM &&
      (quickAvailableFrom?.availableFrom ?? new Date()) <= now;

    const retentionAvailable =
      retentionCount >= RETENTION_MINIMUM &&
      (retentionAvailableFrom?.availableFrom ?? new Date()) <= now &&
      !!quickAttempt; // Must complete Quick first

    return {
      content_id: contentId,
      content_title: content.title,
      quick_assessment: {
        available: quickAvailable,
        available_since: quickAvailable
          ? quickAvailableFrom?.availableFrom
          : null,
        available_at: !quickAvailable
          ? quickAvailableFrom?.availableFrom
          : null,
        question_count: quickCount,
        student_attempt: quickAttempt
          ? {
              completed: true,
              score_percentage: Number(
                quickAttempt.scorePercentage
              ),
              correct_answers: quickAttempt.correctAnswers,
              total_questions: quickAttempt.totalQuestions,
              completed_at: quickAttempt.completedAt,
              attempt_id: quickAttempt.attemptId,
            }
          : null,
      },
      retention: {
        available: retentionAvailable,
        available_at: retentionAvailableFrom?.availableFrom,
        question_count: retentionCount,
        requires_quick_assessment_first: true,
        quick_assessment_completed: !!quickAttempt,
        student_attempt: retentionAttempt
          ? {
              completed: true,
              score_percentage: Number(
                retentionAttempt.scorePercentage
              ),
              correct_answers: retentionAttempt.correctAnswers,
              total_questions: retentionAttempt.totalQuestions,
              completed_at: retentionAttempt.completedAt,
              attempt_id: retentionAttempt.attemptId,
            }
          : null,
      },
    };
  }

  // -------------------------------------------------------
  // GET AVAILABLE QUIZZES (for student to take)
  // -------------------------------------------------------
  async getAvailableQuizzes(
    contentId: string,
    quizType: string,
    studentId: string
  ) {
    const content = await prisma.content.findUnique({
      where: { id: contentId, deletedAt: null },
      select: { title: true, batchId: true },
    });

    if (!content) {
      throw { code: "CONTENT_NOT_FOUND", statusCode: 404 };
    }

    // Verify enrollment
    await this.verifyStudentEnrollment(studentId, content.batchId);

    // Check prerequisite for Retention quiz
    if (quizType === "RETENTION") {
      const quickAttempt = await prisma.quizAttempt.findFirst({
        where: {
          studentId,
          contentId,
          quizType: "QUICK_ASSESSMENT",
        },
      });
      if (!quickAttempt) {
        throw {
          code: "PREREQUISITE_NOT_MET",
          message:
            "You must complete the Quick Assessment quiz before " +
            "taking the Retention quiz.",
          statusCode: 403,
        };
      }
    }

    // Check not already submitted
    const existingAttempt = await prisma.quizAttempt.findFirst({
      where: { studentId, contentId, quizType: quizType as any },
    });

    if (existingAttempt) {
      throw {
        code: "QUIZ_ALREADY_SUBMITTED",
        message: "You have already completed this quiz.",
        statusCode: 409,
        details: {
          completed_at: existingAttempt.completedAt,
          score_percentage: Number(existingAttempt.scorePercentage),
        },
      };
    }

    const minimum =
      quizType === "QUICK_ASSESSMENT"
        ? QUICK_ASSESSMENT_MINIMUM
        : RETENTION_MINIMUM;

    const now = new Date();

    const quizzes = await prisma.quiz.findMany({
      where: {
        contentId,
        quizType: quizType as any,
        generationStatus: "APPROVED",
        availableFrom: { lte: now },
      },
      select: {
        id: true,
        questionText: true,
        options: true,
        cognitiveLevel: true,
      },
      orderBy: { createdAt: "asc" },
    });

    if (quizzes.length < minimum) {
      throw {
        code: "NO_QUIZZES_AVAILABLE",
        message: "No quiz questions are available yet.",
        statusCode: 404,
      };
    }

    // Randomize option order per student
    // Use student ID as seed for consistent randomization
    // (same student gets same order if they reload)
    const questions = quizzes.map((quiz, index) => {
      const options = quiz.options as string[];
      const displayOrder = this.shuffleWithSeed(
        [0, 1, 2, 3],
        `${studentId}-${quiz.id}`
      );

      return {
        quiz_id: quiz.id,
        question_text: quiz.questionText,
        options: displayOrder.map((i) => options[i]),
        display_order: displayOrder,
        question_number: index + 1,
      };
    });

    const estimatedMinutes = Math.ceil(questions.length * 1.5);

    return {
      quiz_session: {
        content_id: contentId,
        content_title: content.title,
        quiz_type: quizType,
        total_questions: questions.length,
        estimated_minutes: estimatedMinutes,
      },
      questions,
    };
  }

  // -------------------------------------------------------
  // SUBMIT QUIZ
  // -------------------------------------------------------
  async submitQuiz(data: {
    content_id: string;
    quiz_type: string;
    attempt_id: string;
    student_id: string;
    started_at: string;
    responses: Array<{
      quiz_id: string;
      selected_display_index: number;
      time_to_answer_seconds: number | null;
    }>;
    display_orders: Record<string, number[]>;
  }) {
    // Idempotency check — if this attempt_id already exists,
    // return the existing result
    const existingAttempt = await prisma.quizAttempt.findUnique({
      where: { attemptId: data.attempt_id },
      include: {
        content: { select: { title: true } },
      },
    });

    if (existingAttempt) {
      return this.buildResultFromAttempt(
        existingAttempt,
        data.attempt_id,
        data.student_id
      );
    }

    // Check if student has already submitted a DIFFERENT attempt
    const existingAttemptSameQuiz = await prisma.quizAttempt.findFirst({
      where: {
        studentId: data.student_id,
        contentId: data.content_id,
        quizType: data.quiz_type as any,
      },
    });

    if (existingAttemptSameQuiz) {
      throw {
        code: "QUIZ_ALREADY_SUBMITTED",
        message: "You have already completed this quiz.",
        statusCode: 409,
        details: {
          completed_at: existingAttemptSameQuiz.completedAt,
          score_percentage: Number(
            existingAttemptSameQuiz.scorePercentage
          ),
        },
      };
    }

    // Fetch all questions for this quiz
    const quizzes = await prisma.quiz.findMany({
      where: {
        id: { in: data.responses.map((r) => r.quiz_id) },
        contentId: data.content_id,
        quizType: data.quiz_type as any,
        generationStatus: "APPROVED",
      },
      select: {
        id: true,
        questionText: true,
        options: true,
        correctOptionIndex: true,
        explanation: true,
        cognitiveLevel: true,
        batchId: true,
      },
    });

    if (quizzes.length !== data.responses.length) {
      const receivedIds = new Set(data.responses.map((r) => r.quiz_id));
      const validIds = new Set(quizzes.map((q) => q.id));
      const missingIds = [...receivedIds].filter(
        (id) => !validIds.has(id)
      );
      throw {
        code: "INCOMPLETE_SUBMISSION",
        message: `Responses received for ${data.responses.length} 
                  questions but ${quizzes.length} valid questions 
                  found.`,
        statusCode: 400,
        details: { missing_quiz_ids: missingIds },
      };
    }

    const quizMap = Object.fromEntries(
      quizzes.map((q) => [q.id, q])
    );

    const batchId = quizzes[0]?.batchId;
    const now = new Date();
    let correctCount = 0;

    // Build response records and calculate scores
    const responseRecords = data.responses.map((response) => {
      const quiz = quizMap[response.quiz_id];
      const displayOrder = data.display_orders[response.quiz_id];

      // Map display index to canonical index
      const canonicalIndex =
        displayOrder[response.selected_display_index];

      const isCorrect = canonicalIndex === quiz.correctOptionIndex;
      if (isCorrect) correctCount++;

      return {
        studentId: data.student_id,
        quizId: response.quiz_id,
        contentId: data.content_id,
        batchId,
        attemptId: data.attempt_id,
        selectedOptionIndex: canonicalIndex,
        isCorrect,
        timeToAnswerSeconds: response.time_to_answer_seconds,
        submittedAt: now,
      };
    });

    const totalQuestions = data.responses.length;
    const scorePercentage = (correctCount / totalQuestions) * 100;

    const startedAt = new Date(data.started_at);
    const timeTakenSeconds = Math.floor(
      (now.getTime() - startedAt.getTime()) / 1000
    );

    // Create all records in a single transaction
    await prisma.$transaction([
      // Create individual response records
      prisma.quizResponse.createMany({
        data: responseRecords,
        skipDuplicates: true,
      }),
      // Create attempt summary record
      prisma.quizAttempt.create({
        data: {
          attemptId: data.attempt_id,
          studentId: data.student_id,
          contentId: data.content_id,
          batchId,
          quizType: data.quiz_type as any,
          totalQuestions,
          correctAnswers: correctCount,
          scorePercentage,
          timeTakenSeconds,
          startedAt,
          completedAt: now,
        },
      }),
    ]);

    // Trigger metrics recalculation for this student
    await metricsQueue.add(
      "RECALCULATE_STUDENT_METRICS",
      {
        student_id: data.student_id,
        batch_id: batchId,
        triggered_by: "QUIZ_SUBMISSION",
      },
      { attempts: 3 }
    );

    // Build result response
    const results = data.responses.map((response) => {
      const quiz = quizMap[response.quiz_id];
      const options = quiz.options as string[];
      const displayOrder = data.display_orders[response.quiz_id];
      const canonicalSelected =
        displayOrder[response.selected_display_index];
      const isCorrect = canonicalSelected === quiz.correctOptionIndex;

      return {
        quiz_id: response.quiz_id,
        question_text: quiz.questionText,
        your_answer: options[canonicalSelected],
        correct_answer: options[quiz.correctOptionIndex],
        is_correct: isCorrect,
        explanation: quiz.explanation,
      };
    });

    return {
      attempt_id: data.attempt_id,
      quiz_type: data.quiz_type,
      content_title:
        quizzes[0] ? "content" : "Unknown",
      score: {
        correct_answers: correctCount,
        total_questions: totalQuestions,
        score_percentage: scorePercentage,
        time_taken_seconds: timeTakenSeconds,
      },
      results,
    };
  }

  // -------------------------------------------------------
  // GET QUIZ HISTORY
  // -------------------------------------------------------
  async getQuizHistory(
    studentId: string,
    requestingUserId: string,
    requestingUserRole: string,
    filters: {
      content_id?: string;
      quiz_type?: string;
      batch_id?: string;
    },
    pagination: { page: number; limit: number }
  ) {
    // Students can only see their own history
    if (
      requestingUserRole === "STUDENT" &&
      studentId !== requestingUserId
    ) {
      throw { code: "PERMISSION_DENIED", statusCode: 403 };
    }

    const whereClause: any = { studentId };
    if (filters.content_id) whereClause.contentId = filters.content_id;
    if (filters.quiz_type) whereClause.quizType = filters.quiz_type;
    if (filters.batch_id) whereClause.batchId = filters.batch_id;

    const [attempts, total] = await Promise.all([
      prisma.quizAttempt.findMany({
        where: whereClause,
        orderBy: { completedAt: "desc" },
        skip: (pagination.page - 1) * pagination.limit,
        take: pagination.limit,
        include: {
          content: { select: { title: true } },
        },
      }),
      prisma.quizAttempt.count({ where: whereClause }),
    ]);

    // Calculate summary
    const allAttempts = await prisma.quizAttempt.findMany({
      where: { studentId, ...(filters.batch_id
        ? { batchId: filters.batch_id }
        : {}) },
      select: {
        scorePercentage: true,
        timeTakenSeconds: true,
      },
    });

    const scores = allAttempts.map((a) =>
      Number(a.scorePercentage)
    );
    const avgScore =
      scores.length > 0
        ? scores.reduce((sum, s) => sum + s, 0) / scores.length
        : 0;

    const totalTimeSeconds = allAttempts.reduce(
      (sum, a) => sum + (a.timeTakenSeconds ?? 0),
      0
    );

    return {
      attempts: attempts.map((a) => ({
        attempt_id: a.attemptId,
        content_id: a.contentId,
        content_title: a.content.title,
        quiz_type: a.quizType,
        score_percentage: Number(a.scorePercentage),
        correct_answers: a.correctAnswers,
        total_questions: a.totalQuestions,
        time_taken_seconds: a.timeTakenSeconds,
        completed_at: a.completedAt,
      })),
      summary: {
        total_quizzes_taken: total,
        average_score: Math.round(avgScore * 10) / 10,
        highest_score:
          scores.length > 0 ? Math.max(...scores) : 0,
        lowest_score:
          scores.length > 0 ? Math.min(...scores) : 0,
        total_time_minutes: Math.floor(totalTimeSeconds / 60),
      },
      pagination: {
        page: pagination.page,
        limit: pagination.limit,
        total,
        total_pages: Math.ceil(total / pagination.limit),
      },
    };
  }

  // -------------------------------------------------------
  // PRIVATE HELPERS
  // -------------------------------------------------------

  private async verifyStudentEnrollment(
    studentId: string,
    batchId: string
  ) {
    const enrollment = await prisma.enrollment.findFirst({
      where: { studentId, batchId, status: "ACTIVE" },
    });

    if (!enrollment) {
      throw {
        code: "NOT_ENROLLED",
        message: "You are not enrolled in this batch",
        statusCode: 403,
      };
    }
  }

  // Deterministic shuffle using a string seed
  // Same student always gets same option order for
  // the same question (consistent across page reloads)
  private shuffleWithSeed(array: number[], seed: string): number[] {
    const result = [...array];
    let hash = 0;

    for (let i = 0; i < seed.length; i++) {
      hash = ((hash << 5) - hash + seed.charCodeAt(i)) | 0;
    }

    for (let i = result.length - 1; i > 0; i--) {
      hash = ((hash << 5) - hash + i) | 0;
      const j = Math.abs(hash) % (i + 1);
      [result[i], result[j]] = [result[j], result[i]];
    }

    return result;
  }

  private async buildResultFromAttempt(
    attempt: any,
    attemptId: string,
    studentId: string
  ) {
    const responses = await prisma.quizResponse.findMany({
      where: { attemptId, studentId },
      include: {
        quiz: {
          select: {
            questionText: true,
            options: true,
            correctOptionIndex: true,
            explanation: true,
          },
        },
      },
      orderBy: { submittedAt: "asc" },
    });

    return {
      attempt_id: attemptId,
      quiz_type: attempt.quizType,
      content_title: attempt.content?.title,
      score: {
        correct_answers: attempt.correctAnswers,
        total_questions: attempt.totalQuestions,
        score_percentage: Number(attempt.scorePercentage),
        time_taken_seconds: attempt.timeTakenSeconds ?? 0,
      },
      results: responses.map((r) => {
        const options = r.quiz.options as string[];
        return {
          quiz_id: r.quizId,
          question_text: r.quiz.questionText,
          your_answer: options[r.selectedOptionIndex],
          correct_answer: options[r.quiz.correctOptionIndex],
          is_correct: r.isCorrect,
          explanation: r.quiz.explanation,
        };
      }),
    };
  }
}
```

```typescript
    }),
```

## 8.1 Error Code Reference

```
NOT_ENROLLED             : 403 — Student not in batch

PREREQUISITE_NOT_MET     : 403 — Quick Assessment not done yet

INVALID_QUIZ_IDS         : 400 — Quiz IDs not valid for this content
QUIZ_NOT_YET_AVAILABLE   : 400 — available_from is in the future
PERMISSION_DENIED        : 403 — Accessing another student's data
```

## 8.2 Display Order Mapping Error Handling

The display_order mapping is a potential source of bugs. If
a display_order array is malformed or missing, the canonical
index calculation will fail. Validate before processing:

```typescript
// In submitQuiz service method
for (const response of data.responses) {
  const displayOrder = data.display_orders[response.quiz_id];

  if (!displayOrder || displayOrder.length !== 4) {
    throw {
      code: "INVALID_DISPLAY_ORDER",
      message: `Missing or invalid display_order for 
                quiz ${response.quiz_id}`,
      statusCode: 400,
    };
  }

  if (
    response.selected_display_index < 0 ||
    response.selected_display_index > 3
  ) {
    throw {
      code: "INVALID_DISPLAY_INDEX",
      message: `selected_display_index must be 0-3`,
      statusCode: 400,
    };
  }

  const canonicalIndex =
    displayOrder[response.selected_display_index];

  if (canonicalIndex < 0 || canonicalIndex > 3) {
    throw {
      code: "INVALID_CANONICAL_INDEX",
      message: `display_order mapping produced invalid 
                canonical index`,
      statusCode: 400,
    };
  }
}
```

## 8.3 Transaction Failure Handling

If the Prisma transaction fails (e.g., database constraint
violation), neither quiz_responses nor quiz_attempt records
are created. The frontend receives a 500 error and the
student can retry submission. The idempotency check ensures
that retrying with the same attempt_id does not create
duplicate records.

# 10. Code Examples

## 10.1 How the Metrics Engine Uses Quiz Response Data

Feature 09 (Metrics Calculation Engine) reads quiz_responses
to calculate six of the nine learning dimensions. Here is
how it fetches and processes the data:

```typescript
// Called by Metrics Calculation Engine (Feature 09)
const getStudentQuizMetrics = async (
  studentId: string,
  batchId: string
) => {
  const responses = await prisma.quizResponse.findMany({
    where: { studentId, batchId },
    include: {
      quiz: {
        select: {
          cognitiveLevel: true,
          quizType: true,
          difficulty: true,
        },
      },
    },
    orderBy: { submittedAt: "asc" },
  });

  const attempts = await prisma.quizAttempt.findMany({
    where: { studentId, batchId },
    orderBy: { completedAt: "asc" },
  });

  // Learning Velocity: score improvement over time
  const weeklyScores = groupAttemptsByWeek(attempts);

  // Conceptual Depth: recall vs application performance
  const recallResponses = responses.filter(
    (r) => r.quiz.cognitiveLevel === "RECALL"
  );
  const applicationResponses = responses.filter(
    (r) =>
      r.quiz.cognitiveLevel === "APPLICATION" ||
      r.quiz.cognitiveLevel === "ANALYSIS"
  );

  const recallScore =
    recallResponses.filter((r) => r.isCorrect).length /
    (recallResponses.length || 1);

  const applicationScore =
    applicationResponses.filter((r) => r.isCorrect).length /
    (applicationResponses.length || 1);

  // Knowledge Retention: retention quiz performance
  const retentionAttempts = attempts.filter(
    (a) => a.quizType === "RETENTION"
  );

  // Error Recovery: performance improvement after poor results
  const poorAttempts = attempts.filter(
    (a) => Number(a.scorePercentage) < 50
  );

  return {
    weekly_scores: weeklyScores,
    recall_score: recallScore * 100,
    application_score: applicationScore * 100,
    retention_attempts: retentionAttempts,
    poor_attempts: poorAttempts,
    all_responses: responses,
  };
};
```

# 11. Performance Optimization

## 11.1 Quiz Submission Throughput

The submit endpoint uses a single Prisma transaction that
creates N quiz_response records (one per question) plus one
quiz_attempt record. For a 15-question quiz, this is 16 database
writes in one transaction. This is fast and acceptable for Phase One.

For Phase Two at scale (100+ students submitting quizzes
simultaneously at the end of a live session), consider:

- Using Prisma's createMany for batch inserts
- Adding a write-through cache for quiz_attempt summaries
- Moving metrics recalculation to a background queue
  (already done via metricsQueue)

## 11.2 Option Shuffle Determinism

The shuffleWithSeed function produces consistent results for
the same student and question, meaning the option order does not
change if the student refreshes the page mid-quiz. This is
intentional — it matches the auto-saved answer indices to the
displayed options correctly.

The seed is student_id + quiz_id, which guarantees:

- Same student always sees same order for a given question
- Different students see different orders for the same question
- No database storage required for the shuffle order
  (calculated on-demand from the seed)

The display_orders map in the submission payload is sent by
the frontend to ensure the backend uses the exact same mapping
that was shown to the student, even if the backend's shuffle
calculation were to differ (e.g., after code changes).

## 11.3 localStorage Quota Management

Each auto-saved quiz state is roughly 1-5KB depending on the
number of questions. A student could have multiple in-progress
quizzes across different content items. Monitor localStorage
usage and implement a cleanup strategy that removes expired
entries on quiz start to prevent quota exhaustion.

```typescript
// Called at the start of startQuiz to clean up expired entries
const cleanupExpiredAutoSaves = () => {
  const keysToDelete: string[] = [];

  for (let i = 0; i < localStorage.length; i++) {
    const key = localStorage.key(i);
    if (!key?.startsWith("quiz_session_")) continue;

    try {
      const data = JSON.parse(localStorage.getItem(key) ?? "{}");
      if (Date.now() > (data.expires_at ?? 0)) {
        keysToDelete.push(key);
      }
    } catch {
      keysToDelete.push(key!);
    }
  }

  keysToDelete.forEach((key) => localStorage.removeItem(key));
};
```
