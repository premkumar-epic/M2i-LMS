# 1. Overview

## 1.1 Database Technology

**Database:** PostgreSQL 15+
**ORM:** Prisma 5+
**Connection Pooling:** Prisma built-in connection pool
**Character Set:** UTF-8
**Timezone:** UTC (all timestamps stored in UTC)

## 1.2 Design Principles

**UUID Primary Keys:** All tables use UUID primary keys
generated by PostgreSQL's gen_random_uuid() function. This
prevents ID enumeration attacks and makes distributed
data management easier in future phases.

**Soft Deletion:** Tables that contain user-generated content
(users, content) use soft deletion via a deleted_at timestamp
column rather than hard deletion. This preserves referential
integrity and historical data.

**Denormalization for Performance:** Several tables store
denormalized foreign keys (e.g., batch_id on quiz_responses)
to avoid expensive JOIN chains on analytics queries. This is
a deliberate trade-off of some write complexity for significant
read performance gains.

**JSONB for Flexible Metadata:** Columns that store variable
structured data (quiz options, notification metadata, alert
metadata) use PostgreSQL's JSONB type which supports
indexing, querying, and validation.

**Audit Timestamps:** Every table includes created_at and
updated_at timestamps. updated_at is automatically maintained
by Prisma's @updatedAt decorator.

## 1.3 Naming Conventions

**Tables:** snake_case, plural noun (users, batches, enrollments)
**Columns:** snake_case, descriptive (student_id, batch_id,
is_published)
**Indexes:** idx_{table}_{column(s)} for regular indexes,
uq_{table}_{column(s)} for unique constraints
**Foreign Keys:** Always named {referenced_table}_id

```sql
-- Recommended PostgreSQL settings for Phase One
-- Add to postgresql.conf

-- Connection pool settings
max_connections = 100
shared_buffers = 256MB

-- Performance settings
work_mem = 16MB
maintenance_work_mem = 64MB

-- WAL settings for reliability
wal_level = replica
```


# 4. Table 01 — Users

## 4.1 Purpose

Central user account table. Every person who accesses the
platform has exactly one record here. All other tables that
store user-specific data reference this table via foreign keys.

## 4.2 SQL Definition

```sql
CREATE TABLE users (
  id              UUID          PRIMARY KEY
                                DEFAULT gen_random_uuid(),
  email           VARCHAR(255)  NOT NULL,
  password_hash   VARCHAR(255)  NOT NULL,
  full_name       VARCHAR(255)  NOT NULL,
  role            VARCHAR(20)   DEFAULT NULL
                                CHECK (role IS NULL OR role IN (
                                  'STUDENT',
                                  'MENTOR',
                                  'ADMIN',
                                  'SUPER_ADMIN'
                                )),
  avatar_url      TEXT          DEFAULT NULL,
  is_active       BOOLEAN       NOT NULL DEFAULT TRUE,
  last_login_at   TIMESTAMP     WITH TIME ZONE DEFAULT NULL,
  created_at      TIMESTAMP     WITH TIME ZONE NOT NULL
                                DEFAULT NOW(),
  updated_at      TIMESTAMP     WITH TIME ZONE NOT NULL
                                DEFAULT NOW(),
  deleted_at      TIMESTAMP     WITH TIME ZONE DEFAULT NULL
);

-- Unique constraint on email
CREATE UNIQUE INDEX uq_users_email
  ON users(email);

-- Index for role-based lookups
CREATE INDEX idx_users_role
  ON users(role)
  WHERE deleted_at IS NULL AND role IS NOT NULL;

-- Index for active user checks
CREATE INDEX idx_users_is_active
  ON users(is_active)
  WHERE deleted_at IS NULL;
```

## 4.3 Column Reference


| Column        | Type         | Nullable | Default           | Description                                 |
| ------------- | ------------ | -------- | ----------------- | ------------------------------------------- |
| id            | UUID         | NO       | gen_random_uuid() | Primary key                                 |
| email         | VARCHAR(255) | NO       | —                | Login identifier, unique                    |
| password_hash | VARCHAR(255) | NO       | —                | bcrypt hash, never plaintext                |
| full_name     | VARCHAR(255) | NO       | —                | Display name                                |
| role          | VARCHAR(20)  | YES      | NULL              | STUDENT, MENTOR, ADMIN, SUPER_ADMIN or NULL |
| avatar_url    | TEXT         | YES      | NULL              | S3/CDN URL to profile image                 |
| is_active     | BOOLEAN      | NO       | TRUE              | FALSE = account deactivated                 |
| last_login_at | TIMESTAMPTZ  | YES      | NULL              | Most recent successful login                |
| created_at    | TIMESTAMPTZ  | NO       | NOW()             | Account creation time                       |
| updated_at    | TIMESTAMPTZ  | NO       | NOW()             | Last modification time                      |
| deleted_at    | TIMESTAMPTZ  | YES      | NULL              | Soft deletion timestamp                     |

## 4.4 Business Rules

- email must be lowercase before storage — enforce at application layer
- password_hash is always bcrypt with 10+ salt rounds
- role = NULL means account created but not yet role-assigned
- deleted_at is set instead of deleting the row to preserve
  all foreign key references
- A SUPER_ADMIN cannot deactivate the last SUPER_ADMIN account

## 4.5 Sample Data

```sql
INSERT INTO users (
  id, email, password_hash, full_name, role, is_active
) VALUES (
  gen_random_uuid(),
  'admin@m2ilms.com',
  '$2a$10$...bcrypt_hash...',
  'Platform Admin',
  'SUPER_ADMIN',
  TRUE
);
```

# 5. Table 02 — RefreshTokens

## 5.1 Purpose

Stores hashed JWT refresh tokens. Never stores the raw token
itself — only the SHA-256 hash. This ensures that even if the
table is compromised, the hashes cannot be used directly.

## 5.2 SQL Definition

```sql
CREATE TABLE refresh_tokens (
  id          UUID          PRIMARY KEY
                            DEFAULT gen_random_uuid(),
  user_id     UUID          NOT NULL
                            REFERENCES users(id)
                            ON DELETE CASCADE,
  token_hash  VARCHAR(255)  NOT NULL,
  expires_at  TIMESTAMP     WITH TIME ZONE NOT NULL,
  created_at  TIMESTAMP     WITH TIME ZONE NOT NULL
                            DEFAULT NOW(),
  revoked_at  TIMESTAMP     WITH TIME ZONE DEFAULT NULL
);

-- Unique constraint on token hash
CREATE UNIQUE INDEX uq_refresh_tokens_hash
  ON refresh_tokens(token_hash);

-- Index for user lookup
CREATE INDEX idx_refresh_tokens_user_id
  ON refresh_tokens(user_id);

-- Index for expiry cleanup
CREATE INDEX idx_refresh_tokens_expires_at
  ON refresh_tokens(expires_at)
  WHERE revoked_at IS NULL;
```

## 5.3 Column Reference


| Column     | Type         | Nullable | Default           | Description                                      |
| ---------- | ------------ | -------- | ----------------- | ------------------------------------------------ |
| id         | UUID         | NO       | gen_random_uuid() | Primary key                                      |
| user_id    | UUID         | NO       | —                | FK → users.id, CASCADE DELETE                   |
| token_hash | VARCHAR(255) | NO       | —                | SHA-256 hash of raw token                        |
| expires_at | TIMESTAMPTZ  | NO       | —                | When this token expires (7 days)                 |
| created_at | TIMESTAMPTZ  | NO       | NOW()             | When token was issued                            |
| revoked_at | TIMESTAMPTZ  | YES      | NULL              | When token was revoked (logout, password change) |

## 5.4 Cleanup Job

Expired and revoked tokens should be cleaned up weekly
to keep this table small:

```sql
-- Delete tokens expired more than 7 days ago
DELETE FROM refresh_tokens
WHERE expires_at < NOW() - INTERVAL '7 days';
```

# 6. Table 03 — LoginAttempts

## 6.1 Purpose

Records every login attempt (success and failure) for rate
limiting and security auditing. Used by the rate limiter
middleware to detect brute force attacks.

## 6.2 SQL Definition

```sql
CREATE TABLE login_attempts (
  id            UUID          PRIMARY KEY
                              DEFAULT gen_random_uuid(),
  email         VARCHAR(255)  NOT NULL,
  ip_address    VARCHAR(45)   NOT NULL,
  success       BOOLEAN       NOT NULL,
  attempted_at  TIMESTAMP     WITH TIME ZONE NOT NULL
                              DEFAULT NOW()
);

-- Composite index for IP-based rate limiting
CREATE INDEX idx_login_attempts_ip_time
  ON login_attempts(ip_address, attempted_at DESC);

-- Composite index for account-based rate limiting
CREATE INDEX idx_login_attempts_email_time
  ON login_attempts(email, attempted_at DESC);
```

## 6.3 Column Reference


| Column       | Type         | Nullable | Default           | Description               |
| ------------ | ------------ | -------- | ----------------- | ------------------------- |
| id           | UUID         | NO       | gen_random_uuid() | Primary key               |
| email        | VARCHAR(255) | NO       | —                | Email used in attempt     |
| ip_address   | VARCHAR(45)  | NO       | —                | Client IP (supports IPv6) |
| success      | BOOLEAN      | NO       | —                | TRUE = successful login   |
| attempted_at | TIMESTAMPTZ  | NO       | NOW()             | When attempt occurred     |

## 6.4 Cleanup Job

```sql
-- Delete attempts older than 30 days
DELETE FROM login_attempts
WHERE attempted_at < NOW() - INTERVAL '30 days';
```

# 7. Table 04 — Batches

## 7.1 Purpose

Represents a student cohort — a group of students going through
a defined curriculum together over a specific time period.
Every content item, quiz, session, and progress record belongs
to a batch.

## 7.2 SQL Definition

```sql
CREATE TABLE batches (
  id          UUID          PRIMARY KEY
                            DEFAULT gen_random_uuid(),
  name        VARCHAR(255)  NOT NULL,
  description TEXT          DEFAULT NULL,
  start_date  DATE          NOT NULL,
  end_date    DATE          NOT NULL,
  status      VARCHAR(15)   NOT NULL DEFAULT 'DRAFT'
                            CHECK (status IN (
                              'DRAFT',
                              'ACTIVE',
                              'COMPLETED',
                              'ARCHIVED'
                            )),
  created_by  UUID          NOT NULL
                            REFERENCES users(id),
  created_at  TIMESTAMP     WITH TIME ZONE NOT NULL
                            DEFAULT NOW(),
  updated_at  TIMESTAMP     WITH TIME ZONE NOT NULL
                            DEFAULT NOW(),

  CONSTRAINT chk_batches_dates
    CHECK (end_date > start_date),

  CONSTRAINT chk_batches_min_duration
    CHECK (end_date >= start_date + INTERVAL '7 days')
);

-- Unique batch names
CREATE UNIQUE INDEX uq_batches_name
  ON batches(name);

-- Index for status filtering
CREATE INDEX idx_batches_status
  ON batches(status)
  WHERE status IN ('DRAFT', 'ACTIVE');

-- Index for date-based status transitions
CREATE INDEX idx_batches_start_date
  ON batches(start_date)
  WHERE status = 'DRAFT';

CREATE INDEX idx_batches_end_date
  ON batches(end_date)
  WHERE status = 'ACTIVE';
```

## 7.3 Column Reference


| Column      | Type         | Nullable | Default           | Description            |
| ----------- | ------------ | -------- | ----------------- | ---------------------- |
| id          | UUID         | NO       | gen_random_uuid() | Primary key            |
| name        | VARCHAR(255) | NO       | —                | Unique display name    |
| description | TEXT         | YES      | NULL              | Optional description   |
| start_date  | DATE         | NO       | —                | When batch begins      |
| end_date    | DATE         | NO       | —                | When batch ends        |
| status      | VARCHAR(15)  | NO       | 'DRAFT'           | Lifecycle status       |
| created_by  | UUID         | NO       | —                | FK → users.id (Admin) |
| created_at  | TIMESTAMPTZ  | NO       | NOW()             | Creation time          |
| updated_at  | TIMESTAMPTZ  | NO       | NOW()             | Last update            |

## 7.4 Status Transition Rules

```
DRAFT → ACTIVE       : Automatic when start_date arrives (nightly job)
                       OR manual by Admin
ACTIVE → COMPLETED   : Automatic when end_date passes (nightly job)
ACTIVE → ARCHIVED    : Manual by Admin only
COMPLETED → ARCHIVED : Manual by Admin only
DRAFT → CANCELLED    : Manual by Admin (if batch never started)
```

# 8. Table 05 — Enrollments

## 8.1 Purpose

Junction table linking students to batches. Each enrollment
record represents one student's membership in one batch.
The status column tracks whether the student is currently
active, withdrawn, or completed.

## 8.2 SQL Definition

```sql
CREATE TABLE enrollments (
  id            UUID          PRIMARY KEY
                              DEFAULT gen_random_uuid(),
  student_id    UUID          NOT NULL
                              REFERENCES users(id),
  batch_id      UUID          NOT NULL
                              REFERENCES batches(id),
  status        VARCHAR(15)   NOT NULL DEFAULT 'ACTIVE'
                              CHECK (status IN (
                                'ACTIVE',
                                'WITHDRAWN',
                                'COMPLETED'
                              )),
  enrolled_at   TIMESTAMP     WITH TIME ZONE NOT NULL
                              DEFAULT NOW(),
  enrolled_by   UUID          NOT NULL
                              REFERENCES users(id),
  withdrawn_at  TIMESTAMP     WITH TIME ZONE DEFAULT NULL,
  withdrawn_by  UUID          DEFAULT NULL
                              REFERENCES users(id),
  completed_at  TIMESTAMP     WITH TIME ZONE DEFAULT NULL
);

-- One enrollment per student per batch
CREATE UNIQUE INDEX uq_enrollments_student_batch
  ON enrollments(student_id, batch_id);

-- Batch member lookups
CREATE INDEX idx_enrollments_batch_id
  ON enrollments(batch_id, status);

-- Student enrollment history
CREATE INDEX idx_enrollments_student_id
  ON enrollments(student_id, status);

-- Partial index for active enrollments (most common query)
CREATE INDEX idx_enrollments_active
  ON enrollments(student_id)
  WHERE status = 'ACTIVE';
```

## 8.3 Column Reference


| Column       | Type        | Nullable | Default           | Description                         |
| ------------ | ----------- | -------- | ----------------- | ----------------------------------- |
| id           | UUID        | NO       | gen_random_uuid() | Primary key                         |
| student_id   | UUID        | NO       | —                | FK → users.id (STUDENT role)       |
| batch_id     | UUID        | NO       | —                | FK → batches.id                    |
| status       | VARCHAR(15) | NO       | 'ACTIVE'          | ACTIVE, WITHDRAWN, COMPLETED        |
| enrolled_at  | TIMESTAMPTZ | NO       | NOW()             | Enrollment creation time            |
| enrolled_by  | UUID        | NO       | —                | FK → users.id (Admin who enrolled) |
| withdrawn_at | TIMESTAMPTZ | YES      | NULL              | When student was withdrawn          |
| withdrawn_by | UUID        | YES      | NULL              | FK → users.id (Admin who withdrew) |
| completed_at | TIMESTAMPTZ | YES      | NULL              | Auto-set when batch completes       |

# 9. Table 06 — BatchMentors

## 9.1 Purpose

Junction table linking mentors to batches. A mentor can be
assigned to multiple batches. A batch can have multiple mentors.
An assigned mentor gains full management access to that batch's
content, quizzes, and sessions.

## 9.2 SQL Definition

```sql
CREATE TABLE batch_mentors (
  id            UUID        PRIMARY KEY
                            DEFAULT gen_random_uuid(),
  batch_id      UUID        NOT NULL
                            REFERENCES batches(id),
  mentor_id     UUID        NOT NULL
                            REFERENCES users(id),
  assigned_at   TIMESTAMP   WITH TIME ZONE NOT NULL
                            DEFAULT NOW(),
  assigned_by   UUID        NOT NULL
                            REFERENCES users(id)
);

-- One assignment per mentor per batch
CREATE UNIQUE INDEX uq_batch_mentors_batch_mentor
  ON batch_mentors(batch_id, mentor_id);

-- Batch mentor lookup
CREATE INDEX idx_batch_mentors_batch_id
  ON batch_mentors(batch_id);

-- Mentor batch lookup
CREATE INDEX idx_batch_mentors_mentor_id
  ON batch_mentors(mentor_id);
```

## 9.3 Column Reference


| Column      | Type        | Nullable | Default           | Description                         |
| ----------- | ----------- | -------- | ----------------- | ----------------------------------- |
| id          | UUID        | NO       | gen_random_uuid() | Primary key                         |
| batch_id    | UUID        | NO       | —                | FK → batches.id                    |
| mentor_id   | UUID        | NO       | —                | FK → users.id (MENTOR role)        |
| assigned_at | TIMESTAMPTZ | NO       | NOW()             | When assignment was created         |
| assigned_by | UUID        | NO       | —                | FK → users.id (Admin who assigned) |

# 10. Table 07 — Content

## 10.1 Purpose

Stores all learning materials — videos, documents, and resource
links. This is the central table for the Content Management
System. Every uploaded video triggers the transcription and
quiz generation pipeline. Content is always scoped to a batch.

## 10.2 SQL Definition

```sql
CREATE TABLE content (
  id                    UUID          PRIMARY KEY
                                      DEFAULT gen_random_uuid(),
  batch_id              UUID          NOT NULL
                                      REFERENCES batches(id),
  uploaded_by           UUID          NOT NULL
                                      REFERENCES users(id),
  title                 VARCHAR(255)  NOT NULL,
  description           TEXT          DEFAULT NULL,
  content_type          VARCHAR(20)   NOT NULL
                                      CHECK (content_type IN (
                                        'VIDEO',
                                        'DOCUMENT',
                                        'RESOURCE',
                                        'LIVE_RECORDING'
                                      )),
  storage_url           TEXT          NOT NULL,
  cdn_url               TEXT          DEFAULT NULL,
  duration_seconds      INTEGER       DEFAULT NULL
                                      CHECK (duration_seconds > 0),
  file_size_bytes       BIGINT        DEFAULT NULL
                                      CHECK (file_size_bytes > 0),
  mime_type             VARCHAR(100)  DEFAULT NULL,
  topic_tags            TEXT[]        NOT NULL DEFAULT '{}',
  learning_objectives   TEXT          DEFAULT NULL,
  transcript            TEXT          DEFAULT NULL,
  transcription_status  VARCHAR(20)   NOT NULL DEFAULT 'PENDING'
                                      CHECK (transcription_status IN (
                                        'PENDING',
                                        'PROCESSING',
                                        'COMPLETE',
                                        'FAILED',
                                        'NOT_REQUIRED'
                                      )),
  transcription_error   TEXT          DEFAULT NULL,
  is_published          BOOLEAN       NOT NULL DEFAULT FALSE,
  sort_order            INTEGER       NOT NULL DEFAULT 0,
  live_session_id       UUID          DEFAULT NULL
                                      REFERENCES live_sessions(id),
  created_at            TIMESTAMP     WITH TIME ZONE NOT NULL
                                      DEFAULT NOW(),
  updated_at            TIMESTAMP     WITH TIME ZONE NOT NULL
                                      DEFAULT NOW(),
  deleted_at            TIMESTAMP     WITH TIME ZONE DEFAULT NULL
);

-- Index for batch content listing (most common query)
CREATE INDEX idx_content_batch_published
  ON content(batch_id, is_published, sort_order)
  WHERE deleted_at IS NULL;

-- Index for mentor content management
CREATE INDEX idx_content_batch_all
  ON content(batch_id, sort_order)
  WHERE deleted_at IS NULL;

-- Index for transcription job queue
CREATE INDEX idx_content_transcription_pending
  ON content(transcription_status, created_at)
  WHERE transcription_status IN ('PENDING', 'PROCESSING');

-- Index for uploader reference
CREATE INDEX idx_content_uploaded_by
  ON content(uploaded_by);
```

## 10.3 Column Reference


| Column               | Type         | Nullable | Default           | Description                               |
| -------------------- | ------------ | -------- | ----------------- | ----------------------------------------- |
| id                   | UUID         | NO       | gen_random_uuid() | Primary key                               |
| batch_id             | UUID         | NO       | —                | FK → batches.id                          |
| uploaded_by          | UUID         | NO       | —                | FK → users.id (mentor)                   |
| title                | VARCHAR(255) | NO       | —                | Display title                             |
| description          | TEXT         | YES      | NULL              | Optional description                      |
| content_type         | VARCHAR(20)  | NO       | —                | VIDEO, DOCUMENT, RESOURCE, LIVE_RECORDING |
| storage_url          | TEXT         | NO       | —                | S3 key or external URL                    |
| cdn_url              | TEXT         | YES      | NULL              | CloudFront delivery URL                   |
| duration_seconds     | INTEGER      | YES      | NULL              | Video duration                            |
| file_size_bytes      | BIGINT       | YES      | NULL              | File size in bytes                        |
| mime_type            | VARCHAR(100) | YES      | NULL              | MIME type                                 |
| topic_tags           | TEXT[]       | NO       | '{}'              | Array of topic strings                    |
| learning_objectives  | TEXT         | YES      | NULL              | Mentor-defined objectives for quiz gen    |
| transcript           | TEXT         | YES      | NULL              | Whisper-generated transcript              |
| transcription_status | VARCHAR(20)  | NO       | 'PENDING'         | Pipeline status                           |
| transcription_error  | TEXT         | YES      | NULL              | Error message if failed                   |

## 11.4 Update Pattern

```sql
-- UPSERT pattern used by backend on every 30-second progress update
INSERT INTO content_access_logs (
  student_id, content_id, batch_id,
  total_watch_time_seconds, last_position_seconds,
  completion_percentage, is_completed
)
VALUES ($1, $2, $3, $4, $5, $6, $7)
ON CONFLICT (student_id, content_id)
DO UPDATE SET
  last_accessed_at = NOW(),
  total_watch_time_seconds = content_access_logs.total_watch_time_seconds + $4,
  last_position_seconds = $5,
  completion_percentage = GREATEST(
    content_access_logs.completion_percentage, $6
  ),
  is_completed = CASE
    WHEN $6 >= 90 THEN TRUE
    ELSE content_access_logs.is_completed
  END,
  access_count = CASE
    WHEN content_access_logs.last_accessed_at < NOW() - INTERVAL '1 hour'
    THEN content_access_logs.access_count + 1
    ELSE content_access_logs.access_count
  END,
  rewatch_count = CASE
    WHEN content_access_logs.is_completed = TRUE
      AND content_access_logs.last_accessed_at < NOW() - INTERVAL '1 hour'
    THEN content_access_logs.rewatch_count + 1
    ELSE content_access_logs.rewatch_count
  END;
```

# 28. Data Retention Policy

## 28.1 Active Data

All data for active batches and their enrolled students is
retained indefinitely while the batch is ACTIVE or COMPLETED.

## 28.2 Archived Data

When a batch is ARCHIVED:

- All student progress records: Retained permanently
- Quiz responses and attempts: Retained permanently
- Content and quizzes: Retained permanently (soft delete only)
- Session recordings: Retained for 90 days after archiving

## 28.3 Automatic Cleanup Jobs


| Data                     | Retention Period    | Cleanup Trigger |
| ------------------------ | ------------------- | --------------- |
| refresh_tokens (expired) | 7 days after expiry | Weekly job      |
| login_attempts           | 30 days             | Daily job       |
| notifications (read)     | 30 days             | Daily job       |
| quiz_generation_logs     | 90 days             | Monthly job     |
| metrics_calculation_logs | 90 days             | Monthly job     |

## 28.4 Cleanup SQL

```sql
-- Weekly: expired refresh tokens
DELETE FROM refresh_tokens
WHERE expires_at < NOW() - INTERVAL '7 days';

-- Daily: old login attempts
DELETE FROM login_attempts
WHERE attempted_at < NOW() - INTERVAL '30 days';

-- Daily: old read notifications
DELETE FROM notifications
WHERE is_read = TRUE
  AND created_at < NOW() - INTERVAL '30 days';

-- Monthly: old generation logs
DELETE FROM quiz_generation_logs
WHERE created_at < NOW() - INTERVAL '90 days';

-- Monthly: old metrics logs
DELETE FROM metrics_calculation_logs
WHERE completed_at < NOW() - INTERVAL '90 days';
```
